## 1) A Wonderful Contest (مسابقة رائعة)

> [!NOTE]
> 
> **الفكرة التقنية (The Concept):**
> 
> المسألة بتطلب مننا نتأكد هل نقدر نحقق أي سكور صحيح من صفر لحد السكور النهائي للمسابقة كلها ولا لأ الفكرة كلها إن علشان نقدر نجيب أي رقم صحيح بدون ما نسيب أي فجوات لازم نكون بنقدر نزود السكور بتاعنا نقطة بنقطة ودا مستحيل يحصل إلا لو عندنا مسألة واحدة على الأقل متقسمة لـ 100 جزء بحيث كل جزء يدينا نقطة واحدة بالظبط لو مفيش مسألة بالشكل دا يبقى أقل زيادة ممكنة هتكون نقطتين أو أكتر وساعتها مستحيل مثلا نجيب سكور قيمته 1
> 
> **القانون الرياضي والمثال:**
> 
> قيمة النقط لكل جزء فرعي بتتحسب بالمعادلة دي
> 
> $$Score = \frac{100}{a_i}$$
> 
> علشان قيمة الـ `Score` تساوي 1 لازم عدد الأجزاء `a_i` يكون 100 مثال لو عندنا مسألة متقسمة 100 جزء كل جزء بنقطة ومسألة تانية متقسمة 20 جزء كل جزء بـ 5 نقط كدا نقدر نجيب السكور من 1 لـ 100 من المسألة الأولى ولما نعوز نجيب السكور 101 مثلا هناخد جزء بـ 5 نقط من المسألة التانية وناخد 96 جزء من الأولى وهكذا الفكرة إن المسألة أم 100 جزء بتشتغل زي الفكة إللي بتسد أي فجوات بين المسائل الكبيرة

> [!TIP]
> 
> **Gap Filling Strategy (استراتيجية سد الفجوات)**
> 
> في مسائل الرياضيات إللي من النوع دا دايما دور على أصغر وحدة بناء `Building Block` إللي هي هنا مسألة بتدينا نقطة واحدة بمجرد ما نلاقيها بنقدر نكون أي رقم إحنا عايزينه زي فكرة العملات بالظبط لو معاك عملة بجنيه هتقدر تدفع أي مبلغ في الدنيا بدون ما تحتاج فكة

> [!IMPORTANT]
> 
> **الـ Complexity:**
> 
> - **Time Complexity:**
>   
>    بتاخد $O(N)$ لأننا بنلف لفة واحدة بس على الـ `Array` علشان ندور على رقم 100 
> - **Space Complexity:**
>   
>    بتاخد $O(N)$ علشان بنخزن المدخلات كلها جوا مصفوفة ديناميكية `vector`

> [!CAUTION]
> 
> **C++ Template:**
> ```cpp
> void solve() {
>   int n; cin >> n;
>   
>   // Overload قراءة المصفوفة بالكامل في سطر واحد بفضل الـ
>   vector<int> a(n); cin >> a;
> 
>  // count التقييم اللحظي باستخدام دالة
> // No غير كدا نطبع Yes لو لقينا رقم 100 مرة واحدة على الأقل نطبع
>   cout << (count(all(a), 100) ? "Yes" : "No") << endl;
> }
> ```

## 2) Artistic Balance Tree (Parity شجرة التوازن والـ)

> [!NOTE]
> 
> **الفكرة التقنية (The Concept):**
> 
> المسألة بتطلب مننا نقلل مجموع العناصر إللي مش متعلم عليها `Unmarked Elements` لأقصى درجة ودا معناه إننا محتاجين نعلم على أكبر عناصر موجبة في الـ `Array` علشان نشيلها من المجموع فكرة المسألة كلها بتتحل لما نركز في عملية التبديل `Swap` إللي بتحصل حوالين المركز `u` بنلاقي إن العنصر في المكان `u - i` بيتبدل مع العنصر في المكان `u + i`
> 
> الملاحظة السحرية هنا إن المكانين دول دايما ليهم نفس الـ `Parity` يعني لو واحد فيهم زوجي التاني لازم يكون زوجي ولو فردي التاني لازم يكون فردي ودا معناه إن عملية التبديل دي مستحيل تنقل عنصر من مكان فردي لمكان زوجي أبدا!
> 
> **القانون الرياضي والمثال:**
> 
> التطابق الزوجي والفردي للأماكن
> 
> $$(u - i) \pmod 2 == (u + i) \pmod 2$$
> 
> وعلشان كدا إحنا بنقسم المصفوفة لمجموعتين مجموعة للعناصر إللي في أماكن فردية ومجموعة للزوجية ونقسم الاستعلامات `Queries` بنفس الطريقة بنرتب كل مجموعة تنازلي علشان ناخد أكبر أرقام موجبة

> [!TIP]
> 
> **The Wasting Trick (خدعة تضييع الحركات)**
> 
> لو عندنا استعلامات `Queries` في مجموعة معينة إحنا **مُجبرين** نعلم على عنصر واحد على الأقل في المجموعة دي حتى لو كان العنصر دا سالب (وهناخد أقل ضرر يعني أكبر رقم سالب فيهم) بس الميزة إننا نقدر "نضيع" باقي الاستعلامات بتاعة نفس المجموعة على نفس العنصر لأن المسألة بتقول إن التعليم على عنصر متعلم عليه قبل كدا مش بيعمل حاجة علشان كدا في الكود بتاعك إنت حطيت الشرط العبقري دا `if(i == 0 || e_o[p][i] > 0)` يعني أول عنصر بناخده إجباري غصب عننا ولو في استعلامات تانية مش بناخدها إلا لو كانت الأرقام موجبة بس علشان منزودش المجموع

> [!IMPORTANT]
> 
> **الـ Complexity:**
> 
> - **Time Complexity:**
>   
>    بتاخد $O(N \log N)$ لأننا بنحتاج نعمل `Sort` للمجموعتين علشان نرتب الأرقام التنازلية ونجيب الأكبر 
> - **Space Complexity:**
>   
>    بتاخد $O(N)$ علشان بنخزن العناصر جوا الـ `Vectors` بتاعة الـ `Even` والـ `Odd` 

> [!CAUTION]
> 
> **C++ Template:**
> ```cpp
> void solve() {
>   int n, m; cin >> n >> m;
> 
>   int cnt {0};
>   // مصفوفة لتخزين عناصر الأماكن الفردية والزوجية
>   vi e_o[2], c(2, 0);
> 
>   // Parity قراءة العناصر وتوزيعها بناء على الـ
>   for(int i = 0; i < n; i++){
>     int x; cin >> x;
>     cnt += x; // المجموع الكلي المبدئي
>     
>     //  e_o[1] و e_o[0] بتوزع العناصر على i % 2
>     e_o[i % 2].push_back(x);
>   }
> 
>   // Parity قراءة الاستعلامات وعد كام استعلام لكل
>   for(int i = 0; i < m; i++){
>     int x; cin >> x;
>     c[(x - 1) % 2]++;
>   }
> 
>   // معالجة كل مجموعة لوحدها
>   for (int p = 0; p < 2; p++) {
>       // ترتيب تنازلي علشان نجيب الأرقام الكبيرة في الأول
>       sort(e_o[p].begin(), e_o[p].end(), greater<int>());
> 
>       // أقصى عدد نقدر نعلم عليه هو عدد الاستعلامات أو حجم المجموعة
>       int lmt = min((int)e_o[p].size(), c[p]);
>       
>       for (int i = 0; i < lmt; i++) {
>           // إجباري ناخد أول عنصر ولو في عناصر تانية بناخد الموجب بس
>           if(i == 0 || e_o[p][i] > 0) cnt -= e_o[p][i];
>       }
>   }
>   
>   cout << cnt << endl;
> }
> ```
