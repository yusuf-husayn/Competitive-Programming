> [!NOTE] 
> # 1) String Length (طول السلسلة النصية)
> لإيجاد عدد الأحرف (طول) سلسلة نصية في زمن تشغيل ثابت $O(1)$.
> 
> **الفكرة العلمية (The Concept):**
>
> في لغة ++C ، كائن السلسلة النصية `std::string` يحتفظ داخلياً بمتغير يسجل حجمه الفعلي (عدد الأحرف). لذلك، عند استدعاء الدالة المدمجة `length()` أو `size()`، لا يقوم البرنامج بالمرور على جميع الأحرف لعدها (وهو ما كان سيستغرق زمناً قدره $O(N)$)، بل يعود بالقيمة المخزنة مسبقاً بشكل مباشر في زمن $O(1)$. 
> 
> 

> [!TIP] 
> **استخدام `getline` مقابل `cin`**
>
> في حال كانت السلسلة النصية في المدخلات (Input) تحتوي على مسافات فارغة (Spaces)، فإن استخدام `cin >> S` سيؤدي إلى قراءة الكلمة الأولى فقط والتوقف عند أول مسافة. لضمان قراءة السطر بأكمله، يجب استخدام `getline(cin, S)`.
>

> [!CAUTION] 
> **C++ Template:**
> ```cpp
> void solve() {
>  string S;
>  // قراءة السطر كاملاً لتفادي فقدان الأحرف بعد المسافات
>  getline(cin, S);
>     
> // طباعة طول السلسلة النصية مباشرة
>  cout << S.length() << "\n";
> }
> ```

> [!NOTE] 
> # 2) Linear Search in 2D Array (البحث الخطي في مصفوفة ثنائية الأبعاد)
> للبحث عن عنصر $X$ داخل مصفوفة ثنائية الأبعاد باستخدام المرور التسلسلي (Sequential Traversal).
> 
> **الفكرة العلمية (The Concept):**
>
> تعتمد هذه الخوارزمية على استكشاف جميع عناصر المصفوفة صَفاً بصف (Row-Major Order) باستخدام حلقتين تكراريتين متداخلتين (Nested Loops). عند العثور على العنصر المطابق للقيمة $X$، يتم طباعة `YES` وإنهاء تنفيذ الدالة فوراً لتوفير وقت المعالجة. إذا انتهت الحلقات التكرارية بالكامل دون العثور على العنصر، فهذا يثبت رياضياً عدم وجوده، وبالتالي يتم طباعة `NO`.
> 
> **التعقيد الخوارزمي (Algorithmic Complexity):**
>
> - **التعقيد الزمني (Time Complexity):** $O(N \times M)$ في أسوأ الحالات (Worst-case scenario) عندما يكون العنصر غير موجود أو يقع في الخلية الأخيرة من المصفوفة.
> - **التعقيد المكاني (Space Complexity):** $O(N \times M)$ نظراً للحاجة لتخزين الشبكة (Grid) بالكامل في الذاكرة.
> 

> [!TIP] 
> **مبدأ الخروج المبكر (Early Exit / Short-Circuiting)**
>
> استخدام الكلمة المحجوزة `;return` (للدوال من نوع `void`) أو `break;` بمجرد إيجاد العنصر المطلوب يُعد من أهم الممارسات القياسية في الCP . هذا المبدأ يقلل من وقت التنفيذ الفعلي (Runtime) ويتجنب استهلاك موارد المعالج في دورات بحث لا طائل منها بعد تحقيق الهدف.
> 

> [!CAUTION] 
> **C++ Template:**
>```cpp
> void solve() {
>     int N, M; cin >> N >> M;
>     int arr[N][M];
> 
> // قراءة المصفوفة
>   for(int i = 0; i < N; i++) {
>     for(int j = 0; j < M; j++) cin >> arr[i][j];
>     }
> 
>     int x; cin >> x;
> 
> // البحث الخطي
>     for(int i = 0; i < N; i++) {
>       for(int j = 0; j < M; j++) {
>        if(arr[i][j] == x) {
>             cout << "YES\n";
>                 return; // خروج مبكر بمجرد العثور على الهدف
>       }
>     }
>   }
>     cout << "NO\n";
> }
> ```

> [!NOTE] 
> # 3) Swapping Min and Max (تبديل العنصرين الأصغر والأكبر)
> للبحث عن أصغر وأكبر عنصر في مصفوفة وتبديل موقعيهما (In-place Swap) ب Time Complexity $O(N)$.
> 
> **الفكرة العلمية (The Concept):**
>
> لإجراء التبديل داخل المصفوفة الأصلية دون الحاجة لإنشاء مصفوفة جديدة، يجب التعامل مع **عناوين الذاكرة (Pointers/Iterators)** الخاصة بالعناصر بدلاً من قيمها المجردة. 
> دوال المكتبة القياسية مثل `std::min_element` و `std::max_element` لا تُرجع قيمة العنصر، بل تُرجع مؤشراً (Pointer) لمكانه في الذاكرة. باستخدام معامل فك المرجعية (Dereference Operator `*`) لدمج هذه المؤشرات مع دالة `std::swap`، يتم تبديل القيم في العناوين الأصلية مباشرة.
> 
> **التعقيد الخوارزمي (Algorithmic Complexity):**
>
> - **التعقيد الزمني (Time Complexity):** $O(N)$ حيث يتم المرور على المصفوفة خطياً للبحث عن القيمتين.
> - **التعقيد المكاني (Space Complexity):** $O(1)$ مساحة إضافية، حيث يتم التبديل في نفس مكان المصفوفة (In-place).
> 

> [!TIP] 
> **CP Pro Tip: The One-Liner Magic**
>
> دمج عمليات البحث والتبديل في سطر واحد `;swap(*max_element(...), *min_element(...))` يُعد من الممارسات المتقدمة (Idiomatic C++) التي تجعل الكود أقصر وأقل عرضة للأخطاء (Clean Code). وإذا أردت أقصى درجات تحسين الأداء (Optimization) لتقليل عدد دورات البحث، يمكنك استخدام `std::minmax_element` للحصول عليهما معاً في لفة واحدة.
> 

> [!CAUTION] 
> **C++ Template:**
> ```cpp
> void solve() {
>     int n; cin >> n;
>     int arr[n];
>     for(int i = 0; i < n; i++) cin >> arr[i];
> 
>     // الوصول لعناوين الذاكرة وتبديل القيم بداخلها مباشرة
>     swap(*max_element(arr, arr + n), *min_element(arr, arr + n));
> 
>     for(int i = 0; i < n; i++) cout << arr[i] << ' ';
>     cout << "\n";
> }
> ```
