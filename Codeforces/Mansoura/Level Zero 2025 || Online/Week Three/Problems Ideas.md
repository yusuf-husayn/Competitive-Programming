## 1) Three Numbers Sum (مجموع الثلاثة أرقام)

> [!NOTE]
> 
> **الفكرة التقنية (The Concept):**
> 
> المسألة بتطلب مننا نعد كل الاحتمالات الممكنة لـ 3 أرقام $X$ و $Y$ و $Z$ بحيث يكون مجموعهم بيساوي $S$ وكل رقم فيهم ميزيدش عن $K$ وميقلش عن الصفر لو فكرنا بالحل المباشر `Brute Force` هنعمل 3 حلقات تكرارية `Nested Loops` بيلفوا على الـ 3 أرقام بس دا هيجيب `Time Limit Exceeded` لأن $K$ بتوصل لـ 3000 و $3000^3$ يعني 27 مليار عملية ودا مستحيل يخلص في ثانية واحدة الفكرة العبقرية هنا إننا محتاجين نلف على رقمين بس هما $X$ و $Y$ وبما إن المجموع الإجمالي $S$ معروف نقدر نستنتج قيمة $Z$ بمعادلة رياضية بسيطة جدا وبكدا نكون شلنا `Loop` كاملة ووفرنا وقت ضخم
> 
> **القانون الرياضي والمثال:**
> 
> بما إننا متأكدين إن $X + Y + Z = S$
> 
> إذن نقدر نحسب $Z$ كدا:
> 
> $$Z = S - (X + Y)$$
> 
> الخطوة الأخيرة هي إننا نتأكد إن الـ $Z$ إللي حسبناها دي ضمن النطاق المسموح يعني مش سالبة ومش أكبر من $K$
> 
> $$0 \le Z \le K$$
> 
> مثال: لو $K = 2$ و $S = 1$ وإحنا بنلف في الـ `Loops` واخترنا $X = 0$ و $Y = 0$ هنحسب $Z = 1 - (0 + 0) = 1$ هل الـ 1 بين الصفر والـ 2؟ أه إذن نزود العداد بتاعنا

> [!TIP]
> 
> **Time Complexity Reduction (تقليل تعقيد الوقت)**
> 
> التكنيك دا من أهم مهارات البرمجة التنافسية دايما خليك فاكر إن لغة ++C بتقدر تنفذ حوالي $10^8$ عملية في الثانية الواحدة فلما نقلل الحل من $O(K^3)$ لـ $O(K^2)$ إحنا نزلنا عدد العمليات من 27 مليار لـ 9 مليون بس ودا بيخلي الكود يطير ويجيب `Accepted` مرتاح جدا

> [!IMPORTANT]
> 
> **الـ Complexity:**
> 
> - **Time Complexity:**
>   
>    بتاخد $O(K^2)$ لأننا بنعمل حلقتين تكرار بس كل واحدة فيهم بتلف $K$ مرة  
> - **Space Complexity:**
>   
>    بتاخد $O(1)$ علشان مش بنحجز أي مصفوفات أو مساحات زيادة في الميموري وبنستخدم متغيرات بسيطة بس 

> [!CAUTION]
> 
> **C++ Template:**
> ```cpp
> void solve() {
>   int K, S; cin >> K >> S;
>   int cnt {0};
> 
>   //بس Y و X بنلف على
>   for(int X = 0; X <= K; X++){
>     for(int Y = 0; Y <= K; Y++){
>       // o(1) رياضيًا في Z بنستنتج
>       int Z = S - (X + Y);
> 
>       // ضمن الحدود المطلوبة Z نتأكد إن
>       if(Z >= 0 && Z <= K) cnt++;
>     }
>   }
>   cout << cnt << endl;
> }
> ```

## 2) Be Fast (المجموع السريع للمربعات)

> [!NOTE]
> 
> **الفكرة التقنية (The Concept):**
> 
> المسألة من اسمها بتطلب مننا نكون سراع في حساب مجموع مربعات الأرقام من 1 لحد رقم معين `n` لو عملنا حلقة تكرارية `Loop` بتلف وتجمع المربعات هتشتغل عادي لأن `n` أخرها $10^5$ بس الأفضل دايما والأسرع في البرمجة التنافسية إننا نستخدم القانون الرياضي المباشر لمتسلسلة مجموع المربعات وبكدا بنطلع الناتج في خطوة واحدة بس
> 
> **القانون الرياضي والمثال:**
> 
> قانون مجموع مربعات الأعداد المتتالية
> 
> $$Sum = \frac{n \times (n + 1) \times (2n + 1)}{6}$$
> 
> مثال لو `n = 3` هنعوض في القانون هيكون 3 مضروبة في 4 مضروبة في 7 والناتج نقسمه على 6 يعني 84 على 6 تدينا 14 ودا فعلا مجموع 1 و 4 و 9

> [!TIP]
> 
> **Data Types Limits (حدود الأرقام)**
> 
> مع إننا بنستخدم قانون رياضي بيخلص في `O(1)` لازم ناخد بالنا من حاجة مهمة جدا وهي إننا بنضرب `n` في نفسها 3 مرات تقريبا فلو `n` قيمتها $10^5$ حاصل الضرب هيعدي $10^{15}$ ودا رقم ضخم جدا مستحيل يتخزن في متغير من نوع `int` العادي علشان كدا استخدام `long long` هنا إجباري علشان نتجنب الـ `Overflow` والحمد لله إنت مظبط دا في التمبلت بتاعك

> [!IMPORTANT]
> 
> **الـ Complexity:**
> 
> - **Time Complexity:**
>   
>    بتاخد $O(1)$ لأننا بنعمل عملية حسابية مباشرة ومفيش أي `Loops` جوا الكود 
> - **Space Complexity:**
>   
>    بتاخد $O(1)$ علشان بنطبع النتيجة مباشرة بدون حجز مساحات زيادة  

> [!CAUTION]
> 
> **C++ Template:**
> ```cpp
> void solve() {
>   int n; cin >> n;
> 
>   // تعويض مباشر في القانون بخطوة واحدة
>   cout << ((n * (n + 1) * (2 * n + 1)) / 6) << endl;
> }
> ```

## 3) 3D (النجوم في الأبعاد الثلاثة)

> [!NOTE]
> 
> **الفكرة التقنية (The Concept):**
> 
> المسألة بتطلب مننا نحسب لكل نجمة كام نجمة تانية بتشاركها نفس الـ `x` أو الـ `y` أو الـ `z` علشان كدا أنسب طريقة لتمثيل بيانات كل نجمة هي إننا نعمل `Struct` بيجمع التلات أبعاد دول مع بعض في مكان واحد بدل ما نعمل 3 مصفوفات منفصلة ونتلغبط علشان القيود صغيرة $n \le 1000$ نقدر نستخدم حلقتين تكرار `Nested Loops` بنمسك كل نجمة ونقارنها بكل النجوم التانية ولو لقينا نجمتين ليهم نفس الإحداثي بنزود العداد بتاعنا بس أهم حاجة نتأكد إننا مش بنقارن النجمة بنفسها عن طريق الشرط `i != j`
> 
> **القانون الرياضي والمثال:**
> 
> بنطبق 3 شروط مستقلين لكل نجمتين $i$ و $j$
> 
> $$X_i == X_j \implies cx++$$
> 
> $$Y_i == Y_j \implies cy++$$
> 
> $$Z_i == Z_j \implies cz++$$
> 
> مثال لو عندنا نجمة إحداثياتها `(1, 2, 3)` ونجمة تانية إحداثياتها `(1, 5, 2)` هنلاقي إن إحداثي `x` متساوي في الاتنين وقيمته 1 إذن هنزود العداد الأول للنجمة الأولى

> [!TIP]
> 
> **CP Pro Tip: O(N) Frequency Optimization**
> 
> مع إن حل الـ $O(N^2)$ بيعدي في الوقت مرتاح جدا علشان الـ $N$ صغيرة بس لو كانت الـ $N$ بتوصل لـ $10^5$ الحل دا هيجيب `Time Limit Exceeded` علطول ساعتها الحل الأمثل هو إننا نستخدم مصفوفات تكرار `Frequency Arrays` أو `Maps` نعد فيها تكرار كل إحداثي واحنا بنقرأ المدخلات وفي الأخر الإجابة لكل نجمة هتكون التكرار الكلي للإحداثي بتاعها ناقص 1 (علشان بنشيل النجمة نفسها من العد) التكنيك دا بيخلي الكود يخلص في $O(N)$ بس وبيكون أسرع بكتير

> [!IMPORTANT]
> 
> **الـ Complexity:**
> 
> - **Time Complexity:**
>   
>    بتاخد $O(N^2)$ لأننا بنعمل حلقتين تكرار وبنقارن كل نجمة بكل النجوم التانية  
> - **Space Complexity:**
>   
>    بتاخد $O(N)$ علشان بنخزن بيانات كل النجوم جوا مصفوفة واحدة من نوع `Struct` 

> [!CAUTION]
> 
> **C++ Template:**
> ```cpp
> struct star{
>   int x, y, z;
> };
> 
> void solve() {
>   int n; cin >> n;
>   
>   star arr[n];
> 
>   for(int i = 0; i < n; i++) cin >> arr[i].x >> arr[i].y >> arr[i].z;
> 
>   for(int i = 0; i < n; i++){
>     int cx {0}, cy {0}, cz {0};
>     
>     for(int j = 0; j < n; j++){
>       // بنتجاهل النجمة لو بنقارنها بنفسها
>       if(i == j) continue;
> 
>       // مقارنة الإحداثيات وزيادة العدادات
>       if(arr[i].x == arr[j].x) cx++;
>       if(arr[i].y == arr[j].y) cy++;
>       if(arr[i].z == arr[j].z) cz++;
>     }
>     
>     cout << cx << ' ' << cy << ' ' << cz << endl;
>   }
> }
> ```

## 4) Sort Rectangles (ترتيب المستطيلات)

> [!NOTE]
> 
> **الفكرة التقنية (The Concept):**
> 
> المسألة بتطلب مننا نحسب مساحة شوية مستطيلات ونرتبهم تصاعديا حسب المساحة دي ونطبع رقم المستطيل الأصلي (الـ `Index` بتاعه) مع مساحته علشان المستطيل ليه 4 نقط (يعني 8 أرقام `X` و `Y`) مش محتاجين قوانين هندسية معقدة علشان نجيب الطول والعرض إحنا ببساطة بنجيب أكبر `X` ونطرح منها أصغر `X` كدا جبنا الطول ونعمل نفس الكلام مع الـ `Y` علشان نجيب العرض وبما إننا محتاجين نحتفظ بالترتيب الأصلي لكل مستطيل حتى بعد ما نعمل `Sort` أحسن طريقة هي إننا نستخدم `Struct` يلم كل بيانات المستطيل في بلوك واحد زي ما إنت عملت بالظبط
> 
> **القانون الرياضي والمثال:**
> 
> بنحسب الأبعاد والمساحة بالمعادلات دي
> 
> $$Length = \max(X) - \min(X)$$
> 
> $$Width = \max(Y) - \min(Y)$$
> 
> $$Area = Length \times Width$$
> 
> مثال لو نقط المستطيل هي `(10, 7)` و `(3, 7)` و `(10, 6)` و `(3, 6)`
> 
> أكبر `X` هي 10 وأصغر `X` هي 3 إذن الطول بيساوي 7 أكبر `Y` هي 7 وأصغر `Y` هي 6 إذن العرض بيساوي 1 المساحة هتكون $7 \times 1 = 7$

> [!TIP]
> 
> **Custom Comparator (دالة المقارنة المخصصة)**
> 
> استخدامك لدالة مقارنة منفصلة إللي إنت مسميها `cmf` دي مهارة احترافية جدا بتعلم دالة `sort` الجاهزة إزاي ترتب الـ `Structs` بتاعتنا هنا إحنا بنقولها رتبي بناء على المساحة من الصغير للكبير عن طريق الشرط `f.area < s.area` ودا بيخلي الكود منظم جدا ومقري

> [!IMPORTANT]
> 
> **الـ Complexity:**
> 
> - **Time Complexity:**
>   
>    القراءة وحساب المساحات بياخد $O(N)$ بس دالة `sort` بتاخد $O(N \log N)$ إذن الوقت الإجمالي هو $O(N \log N)$ ودا وقت ممتاز جدا ومناسب لعدد $10^5$
> - **Space Complexity:**
>   
>    بتاخد $O(N)$ علشان بنخزن بيانات كل المستطيلات جوا مصفوفة الـ `Struct`  

> [!CAUTION]
> 
> **C++ Template:**
> ```cpp
> struct Rectangle{
>   int idx, area;
> };
> 
> // دالة المقارنة علشان نرتب تصاعديا بالمساحة
> bool cmf(Rectangle f, Rectangle s){
>   return f.area < s.area;
> }
> 
> void solve() {
>   int n; cin >> n;
>   
>   Rectangle arr[n];
> 
>   for(int i = 0; i < n; i++){
>     int X1, Y1, X2, Y2, X3, Y3, X4, Y4; 
>     cin >> X1 >> Y1 >> X2 >> Y2 >> X3 >> Y3 >> X4 >> Y4;
> 
>     int max_X = max({X1, X2, X3, X4});
>     int min_X = min({X1, X2, X3, X4});
>     int max_Y = max({Y1, Y2, Y3, Y4});
>     int min_Y = min({Y1, Y2, Y3, Y4});
> 
>     int length = max_X - min_X, width = max_Y - min_Y;
>     int area = length * width;
> 
>     arr[i] = {i + 1, length, width, area};
>   }
> 
>   //لدالة الترتيب Vector بتوع ال Iterators بنبعت ال
>   sort(arr, arr + n, cmf);
> 
>   for(int i = 0; i < n; i++){
>     cout << arr[i].idx << ' ' << arr[i].area << endl;
>   }
> }
> ```

## 5) Weird Function1 (الدالة العجيبة)

> [!NOTE]
> 
> **الفكرة التقنية (The Concept):**
> 
> المسألة بتطلب مننا نترجم دالة رياضية بسيطة لكود برمجي وننفذ معادلة معينة بتعتمد على استدعاء الدالة دي جوا نفسها أكتر من مرة `Nested Function Calls` الموضوع هنا مجرد تطبيق مباشر على استخدام الـ `Functions` في لغة `C++` بدل ما نكتب المعادلة المعقدة دي كلها يدوي ونفك الأقواس بنعمل دالة صغيرة اسمها `f` بتاخد رقم وتضربه في 2 وتجمع عليه 3 وبعدين بنستدعيها في الـ `main` أو الـ `solve` بالشكل المطلوب بالظبط
> 
> **القانون الرياضي والمثال:**
> 
> الدالة الأساسية هي
> 
> $$f(x) = 2x + 3$$
> 
> والمعادلة المطلوبة هي
> 
> $$Result = f(f(f(x))) + 2 \times f(x \times f(x))$$
> 
> مثال لو $x = 1$:
> 
> أولا $f(1) = 2(1) + 3 = 5$
> 
> الجزء الأول $f(f(f(1)))$ هيتحول لـ $f(f(5))$ يعني $f(13)$ إللي هي هتساوي 29 الجزء التاني $2 \times f(1 \times f(1))$ هيتحول لـ $2 \times f(1 \times 5)$ يعني $2 \times 13$ إللي هي هتساوي 26 المجموع النهائي هيكون $29 + 26 = 55$

> [!TIP]
> 
> **Modular Programming (البرمجة المجزأة)**
> 
> لما بنقسم الكود لدوال صغيرة `Functions` زي دالة `f(int n)` دا بيخلي الكود `Clean` ومقري جدا وبيقلل احتمالية إننا نغلط في الأقواس أو الأولويات الحسابية تخيل لو كنا حاولنا نكتب المعادلة الطويلة دي كلها كسطر واحد رياضي من غير دالة كانت هتبقى لغبطة كبيرة جدا وصعب نعملها `Trace` لو طلعت ناتج غلط

> [!IMPORTANT]
> 
> **الـ Complexity:**
> 
> - **Time Complexity:**
>   
>    بتاخد $O(1)$ لأننا بنعمل استدعاء لدالة بتنفذ عملية حسابية بسيطة عدد ثابت من المرات ومفيش أي `Loops`
> - **Space Complexity:**
>   
>    بتاخد $O(1)$ علشان مش بنحجز أي مساحات في الميموري غير متغيرات بسيطة جدا واستدعاء الدالة مبيعملش ضغط على الـ `Call Stack`

> [!CAUTION]
> 
> **C++ Template:**
> ```cpp
> int f(int n){
>   return (2 * n + 3);
> }
> // الدالة المنفصلة علشان ننفذ المعادلة بسهولة
> void solve() {
>   int x; cin >> x;
> 
>   // تطبيق مباشر للمعادلة المطلوبة باستخدام استدعاء الدالة
>   cout << f(f(f(x))) + 2 * f(x * f(x)) << endl;
> }
> ```

## 6) Weird Function 2 (دالة النصوص العجيبة)

> [!NOTE]
> 
> **الفكرة التقنية (The Concept):**
> 
> المسألة دي بتعتبر الجزء التاني والتطور لمسألة الدالة العجيبة إللي فاتت بس المرة دي بنتعامل مع نصوص `Strings` بدل الأرقام الفكرة كلها إننا محتاجين نعمل دالة `Function` بتاخد رقم `a` ونصين `s1` و `s2` ولو الرقم دا بيساوي 1 بتلزق النصين في بعض بالترتيب الطبيعي ولو بيساوي صفر بتعكس الترتيب وتلزق التاني في الأول وبعدين بنستدعي الدالة دي جوا بعضها بشكل متداخل `Nested Calls` علشان نطلع الناتج النهائي للمسألة
> 
> **القانون الرياضي والمثال:**
> 
> شروط الدالة زي ما المسألة طالبة بالظبط
> 
> $$f(1, s_1, s_2) = s_1 + s_2$$
> 
> $$f(0, s_1, s_2) = s_2 + s_1$$
> 
> مثال لو عندنا الدالة $f(0, \text{"a"}, \text{"b"})$ النتيجة هتكون النص معكوس يعني `ba` ولو استدعينا $f(1, \text{"a"}, \text{"b"})$ النتيجة هتفضل بالترتيب يعني `ab` بنمشي ورا استدعاءات الدالة من الأقواس الداخلية للخارجية لحد ما نكون النص الطويل

> [!TIP]
> 
> **String Concatenation Power (قوة دمج النصوص)**
> 
> دمج النصوص في لغة `C++` باستخدام علامة `+` بيسهل علينا جدا كتابة اللوجيك بتاع المسألة دي بدل ما نعمل `Loops` معقدة أو نتلغبط في تفريعات كتير إحنا بنعمل دالة صغيرة جدا بتستخدم المعامل الثلاثي `Ternary Operator` علشان تختار الترتيب الصح وترجع النص الجاهز فورا دا بيخلي الكود `Clean` جدا وبيخلص المسألة في سطرين

> [!IMPORTANT]
> 
> **الـ Complexity:**
> 
> - **Time Complexity:**
>   
>    بتاخد $O(|S|)$ لأن عمليات الدمج بتاخد وقت بيتناسب مع طول النص المدموج وبما إن طول النصوص في المسألة دي صغير جدا 32 حرف فالوقت الإجمالي يعتبر سريع جدا ومقارب لـ $O(1)$ 
> - **Space Complexity:**
>   
>    بتاخد $O(|S|)$ علشان بنحجز مساحات في الميموري لكل نص جديد بيتبني من نتيجة الاستدعاءات المتداخلة دي 

> [!CAUTION]
> 
> **C++ Template:**
> ```cpp
> string f(int a, string st1, string st2){
>   return (a == 1) ? (st1 + st2) : (st2 + st1);
> }
> > // دالة ذكية بسطر واحد لدمج النصوص بناء على قيمة a
> void solve() {
>   string s1, s2; cin >> s1 >> s2;
> 
>   // استدعاء مباشر ومتداخل للمعادلة المطلوبة بالظبط
>   cout << f(1, f(1, f(0, s1, s2) + s1, s2), s1 + f(0, s1 + s2, f(1, s2, s1))) << endl;
> }
> ```

## 7) Vlad and the Best of Five (فلاد والأفضل من خمسة)

> [!NOTE]
> 
> **الفكرة التقنية (The Concept):**
> 
> المسألة بتدينا نص `String` طوله 5 حروف بالظبط مكون من حروف `A` و `B` بس والمطلوب ببساطة إننا نعرف مين الحرف إللي اتكرر أكتر من التاني الفكرة هنا إن طول النص دايما فردي ودا معناه إن مستحيل يحصل تعادل أبدا ودايما في حرف كسبان فإحنا بنعد تكرار حرف `A` وتكرار حرف `B` ونقارنهم ببعض ونطبع الأغلبية
> 
> **القانون الرياضي والمثال:**
> 
> شرط الفوز المباشر
> 
> $$Count(A) > Count(B) \implies A$$
> 
> $$Count(A) < Count(B) \implies B$$
> 
> مثال لو عندنا النص `ABABB` هنعد حرف `A` هنلاقيه اتكرر مرتين وهنعد حرف `B` هنلاقيه اتكرر 3 مرات بما إن 3 أكبر من 2 إذن الحرف `B` هو الأغلبية ونطبعه

> [!TIP]
> 
> **STL Power (قوة الدوال الجاهزة)**
> 
> استخدام دالة `count` الجاهزة مع المعامل الثلاثي `Ternary Operator` بيخلي الكود `Clean` ومقري جدا وبيخلص اللوجيك كله في سطر واحد سريع بدل ما نعمل `Loop` ونعرف متغيرات للعدادات ونمشي على الحروف حرف حرف بإيدينا

> [!IMPORTANT]
> 
> **الـ Complexity:**
> 
> - **Time Complexity:**
>   
>    بتاخد $O(1)$ لأننا بنلف على 5 حروف بس ودا رقم ثابت مش بيتغير مع كبر حجم المدخلات فبتعتبر سريعة جدا 
> - **Space Complexity:**
>   
>    بتاخد $O(1)$ علشان بنخزن الكلمة الصغيرة دي جوا المتغير `String` ومفيش مساحات زيادة بتتحجز 

> [!CAUTION]
> 
> **C++ Template:**
> ``` cpp
> void solve() {
>   string s; cin >> s;
> 
>   // سطر واحد بيعد ويقارن ويطبع النتيجة اللحظية
>   cout << ((count(all(s), 'A') > count(all(s), 'B')) ? 'A' : 'B') << endl;
> }
> ```

## 8) Highest Y (Y أعلى نقطة على محور)

> [!NOTE]
> 
> **الفكرة التقنية (The Concept):**
> 
> المسألة بتطلب مننا نرتب مجموعة من النقط بناء على الإحداثي `Y` بتاعهم ترتيب تنازلي يعني من الكبير للصغير ونطبعهم بما إن كل نقطة ليها بعدين `X` و `Y` فبدل ما نعمل مصفوفتين ونتلغبط بنعمل `Struct` يلمهم هما الاتنين جوا كيان واحد ونربطهم ببعض بعد كدا بنستخدم دالة `sort` الجاهزة بس بنفهمها إننا عايزين نرتب بناء على الـ `Y` بس وفي اتجاه تنازلي عن طريق دالة مقارنة مخصصة
> 
> **القانون الرياضي والمثال:**
> 
> شرط الترتيب التنازلي في دوال المقارنة بيكون كدا
> 
> $$P_1.Y > P_2.Y$$
> 
> مثال لو عندنا النقطتين `(10, 30)` و `(20, 60)` دالة المقارنة هتشوف إن 60 أكبر من 30 فهتجيب النقطة التانية في الأول ويكون الترتيب النهائي في الطباعة هو `(20, 60)` وبعدين `(10, 30)`

> [!TIP]
> 
> **Custom Comparator (دالة المقارنة المخصصة)**
> 
> علشان نرتب الـ `Struct` براحتنا بنعمل دالة صغيرة اسمها `cmf` بترجع قيمة من نوع `bool` لو كتبنا `return a.Y > b.Y` كدا بنقول للكومبايلر حط العنصر إللي الـ `Y` بتاعته أكبر في الأول ودا بيعمل ترتيب تنازلي ولو كنا عايزين ترتيب تصاعدي كنا هنعكس العلامة لـ `<` التكنيك دا هو أساس التعامل مع ترتيب البيانات المعقدة جوا البرمجة التنافسية

> [!IMPORTANT]
> 
> **الـ Complexity:**
> 
> - **Time Complexity:**
>   
>    بتاخد $O(N \log N)$ لأننا بنستخدم دالة `sort` إللي بتعتمد على خوارزميات سريعة زي `IntroSort` 
> - **Space Complexity:**
>   
>    بتاخد $O(N)$ علشان بنخزن النقط كلها جوا مصفوفة  

> [!CAUTION]
> 
> **C++ Template:**
> ```cpp
> struct point{
>   int X, Y;
> };
> 
> // دالة المقارنة للترتيب التنازلي بناء على Y
> bool cmf(point& a, point& b){
>   return a.Y > b.Y;
> }
> 
> void solve() {
>   int n; cin >> n;
>   point a[n];
> 
>   for(int i = 0; i < n; i++) cin >> a[i].X >> a[i].Y;
> 
>   sort(a, a + n, cmf);
>   
>   for(int i = 0; i < n; i++) cout << a[i].X << ' ' << a[i].Y << endl;
> }
> ```

## 9) Increasing (الترتيب التصاعدي الصارم)

> [!NOTE]
> 
> **الفكرة التقنية (The Concept):**
> 
> المسألة بتطلب مننا نحدد هل نقدر نعيد ترتيب المصفوفة بحيث تكون تصاعدية بشكل صارم `Strictly Increasing` ولا لأ معنى كلمة "تصاعدي صارم" إن كل رقم لازم يكون أكبر تماما من الرقم إللي قبله ودا مستحيل يحصل لو المصفوفة فيها أي أرقام متكررة لأنهم لو جم جنب بعض هيبقوا متساويين مش أكبر من بعض إذن الفكرة كلها بتتلخص في إننا نكتشف هل في تكرار في المصفوفة ولا لأ أبسط طريقة هي إننا نرتب المصفوفة `Sort` علشان الأرقام المتشابهة تيجي جنب بعضها وبعدين نلف عليها بـ `Loop` ونقارن كل رقم باللي قبله لو لقينا رقمين بيساووا بعض يبقى مستحيل نحقق الشرط ونطبع `NO`
> 
> **القانون الرياضي والمثال:**
> 
> شرط الرفض بعد الترتيب
> 
> $$A[i] == A[i - 1] \implies \text{NO}$$
> 
> مثال لو المصفوفة `[8, 7, 1, 3, 4]` بعد الترتيب هتبقى `[1, 3, 4, 7, 8]` مفيش أي رقم بيساوي إللي قبله إذن نطبع `YES`
> 
> مثال تاني لو المصفوفة `[1, 1, 1, 1]` بعد الترتيب هتفضل زي ما هي ولما نقارن هنلاقي إن الـ 1 بيساوي الـ 1 إذن نطبع `NO`

> [!TIP]
> 
> **Data Structures Trick (خدعة الـ Set)**
> 
> مع إن حلك باستخدام الترتيب `Sort` ممتاز وسريع جدا بس في تكنيك تاني مشهور جدا في البرمجة التنافسية لمسائل التكرار وهو استخدام الـ `set` لأن هيكل البيانات دا مش بيقبل تكرار جواه فلو قرينا الأرقام وحطيناها كلها جوا `set` وبعدين قارنا حجم الـ `set` بحجم المصفوفة الأصلي `n` لو طلعوا بيساووا بعض يبقى مفيش تكرار ونطبع `YES` الخدعة دي بتوفر كتابة الـ `Loop` وبتحل المسألة في سطرين

> [!IMPORTANT]
> 
> **الـ Complexity:**
> 
> - **Time Complexity:**
>   
>    بتاخد $O(N \log N)$ لأننا بنستخدم دالة `sort` لترتيب المصفوفة والـ `Loop` إللي بعدها بتاخد $O(N)$ بس الترتيب هو الأبطأ فبيسيطر على التعقيد الكلي
> - **Space Complexity:**
>   
>    بتاخد $O(N)$ علشان بنخزن العناصر جوا مصفوفة الـ `Vector` 

> [!CAUTION]
> 
> **C++ Template:**
> ```cpp
> void solve() {
>   int n; cin >> n;
>   vi a(n); cin >> a;
> 
>   // بنرتب المصفوفة علشان الأرقام المتشابهة تيجي جنب بعض
>   sort(all(a));
>   bool inc = true;
> 
>   // بنلف من العنصر التاني ونقارنه باللي قبله
>   for(int i = 1; i < n; i++){
>     if(a[i] == a[i - 1]){
>       inc = false;
>       break; // بمجرد ما نلاقي تكرار بنوقف البحث علشان نوفر وقت
>     }
>   }
>   
>   cout << ((inc) ? "YES" : "NO" ) << endl;
> }
> ```

## 10) Be Fast 2 (العد السريع للأرقام)

> [!NOTE]
> 
> **الفكرة التقنية (The Concept):**
> 
> المسألة بتطلب مننا نعد الأرقام في النطاق من $L$ لحد $R$ إللي **مش** بتقبل القسمة على $N$
> 
> لو فكرنا نعمل حلقة تكرارية `Loop` تلف من $L$ لـ $R$ وتختبر كل رقم هتجيب `Time Limit Exceeded` علطول لأن الأرقام بتوصل لمليار يعني المعالج هياخد ثواني كتير علشان يخلصها علشان كدا الفكرة كلها بتعتمد على الرياضيات إحنا بنحسب إجمالي عدد الأرقام في النطاق دا وبعدين بنحسب كام رقم بيقبل القسمة على $N$ ونطرحهم من الإجمالي
> 
> **القانون الرياضي والمثال:**
> 
> 1. بنحسب إجمالي الأرقام في النطاق
>     
>     $$Total = R - L + 1$$
>     
> 2. بنحسب عدد الأرقام إللي بتقبل القسمة على $N$ في النطاق دا عن طريق إننا نجيب إللي بيقبلوا القسمة من 1 لحد $R$ ونطرح منهم إللي بيقبلوا القسمة من 1 لحد $L - 1$
>     
>     $$Divisible = \lfloor \frac{R}{N} \rfloor - \lfloor \frac{L - 1}{N} \rfloor$$
>     
> 3. النتيجة النهائية هتكون
>     
>     $$Result = Total - Divisible$$
>     
> 
> مثال لو $L = 5$ و $R = 12$ و $N = 3$
> 
> إجمالي الأرقام $12 - 5 + 1 = 8$ أرقام الأرقام إللي بتقبل على 3 من 1 لـ 12 هما $12 / 3 = 4$ أرقام الأرقام إللي بتقبل على 3 من 1 لـ 4 هما $4 / 3 = 1$ رقم واحد إذن إللي بيقبلوا القسمة في النطاق بتاعنا هما $4 - 1 = 3$ أرقام (إللي هما 6 و 9 و 12) نطرحهم من الإجمالي يبقى $8 - 3 = 5$ أرقام مش بيقبلوا القسمة

> [!TIP]
> 
> **Prefix Sum Concept (مفهوم المجاميع التراكمية)**
> 
> الخدعة بتاعت إننا نطرح من $1 \to R$ ناقص $1 \to L-1$ دي مستوحاة من فكرة الـ `Prefix Sum` ودي مهارة أساسية جدا في الـ `Competitive Programming` بتخلينا نقدر نحسب أي حاجة جوا فترة معينة في زمن $O(1)$ مهما كان حجم الفترة كبير

> [!IMPORTANT]
> 
> **الـ Complexity:**
> 
> - **Time Complexity:**
>   
>    بتاخد $O(1)$ لأننا بنعمل مجرد عمليات حسابية مباشرة جمع وطرح وقسمة ومفيش أي `Loops` 
> - **Space Complexity:**
>   
>    بتاخد $O(1)$ علشان بنخزن النتيجة جوا متغيرات عادية جدا ومش بنحجز أي مصفوفات 

> [!CAUTION]
> 
> **C++ Template:**
> ```cpp
> void solve() {
>   int L, R, N; cin >> L >> R >> N;
> 
>   // حساب عدد الأرقام الكلي
>   int tNum = R - L + 1;
>   
>   // حساب عدد الأرقام إللي بتقبل القسمة في النطاق
>   int divCnt = (R / N) - ((L - 1) / N);
> 
>   // طباعة الأرقام إللي مش بتقبل القسمة
>   cout << tNum - divCnt << endl;
> }
> ```

## 11) Empedocles Problem (مشكلة إيمبيدوكليس)

> [!NOTE]
>  **الفكرة التقنية (The Concept):**
>   المسألة بسيطة جدا وبتطلب مننا نتأكد هل كل العناصر الـ $N$ ليهم نفس الخصائص الـ 4 بالظبط ولا لأ. بدل ما نحفظ كل المدخلات في مصفوفات ونتعب الميموري، إحنا بنقرأ أول 4 كلمات للعنصر الأول ونعتبرهم "المرجع" بتاعنا أو الـ `Reference`. بعد كدا بنعمل حلقة تكرارية `Loop` تلف على باقي العناصر، وكل مرة نقرأ 4 كلمات جداد ونقارنهم بالمرجع الأساسي. لو لقينا أي اختلاف في أي كلمة من الأربعة، بنخلي متغير الـ `bool` اللي اسمه `same` يساوي `false`.
> 
> **القانون الرياضي والمثال:**
> 
> شرط التطابق المنطقي لكل عنصر $i$:
> 
> $$a_i == a_1 \text{ and } b_i == b_1 \text{ and } c_i == c_1 \text{ and } d_i == d_1$$
> 
> لو الشرط دا متحققش في أي لحظة، بنطبع "you were right"، ولو فضل متحقق للآخر بنطبع "Empedocles was right".

> [!TIP] 
> **On-the-fly Processing (المعالجة اللحظية)**
>  الطريقة اللي إنت حليت بيها دي ممتازة جدا لأنك معملتش `vector<string>` ولا خزنت المدخلات كلها في مصفوفة الأول. المعالجة اللحظية دي (إنك تقرأ وتقارن وترمي القيمة القديمة وتشوف اللي بعدها) بتخلي استهلاك الذاكرة بتاعك قليل جدا ودي مهارة ممتازة في الـ `Competitive Programming`.

> [!IMPORTANT]
> 
> **الـ Complexity:**
> 
> - **Time Complexity:**
>   
>    بتاخد $O(N)$ لأننا بنلف لفة واحدة بس على المدخلات. 
> - **Space Complexity:**
>   
>    بتاخد $O(1)$ علشان بنستخدم متغيرات بسيطة جدا لتخزين الكلمات الحالية والمرجعية ومش بنحجز أي مصفوفات. 

> [!CAUTION]
> 
> **C++ Template:**
> ```cpp
> void solve() {
>   int n; cin >> n;
>   
>   // قراءة العنصر الأول كمرجع أساسي
>   string a1, b1, c1, d1; cin >> a1 >> b1 >> c1 >> d1;
> 
>   bool same = true;
>   
>   // اللف على باقي العناصر ومقارنتها بالمرجع
>   for(int i = 1; i < n; i++){
>     string a, b, c, d; cin >> a >> b >> c >> d;
> 
>     // لو في أي اختلاف بنغير حالة التطابق
>     if(a != a1 || b != b1 || c != c1 || d != d1) same = false;
>   }
>   
>   // طباعة النتيجة بناء على المتغير
>   cout << ((same) ? "Empedocles was right" : "you were right") << endl;
> }
> ```

## 12) Dragons (التنانين)

> [!NOTE]
> 
> **الفكرة التقنية (The Concept):**
> 
> المسألة بتطلب مننا نحدد هل نقدر نهزم كل التنانين ولا لأ وإحنا نقدر نختار ترتيب القتال براحتنا بما إن الفوز على أي تنين بيزود قوتنا ومفيش أي تنين بينقص القوة دي إذن أفضل استراتيجية `Greedy` هي إننا نحارب التنانين الأضعف الأول علشان نزود قوتنا بالبونص بتاعهم ودا هيخلينا نقدر نواجه التنانين الأقوى بعدين لو مشينا بالترتيب دا وقابلنا تنين قوته أكبر من أو تساوي قوتنا يبقى مستحيل نكمل الجيم ونطبع `NO`
> 
> **القانون الرياضي والمثال:**
> 
> قانون القتال وتحديث القوة بيكون بالشكل دا
> 
> $$S > X_i \implies S = S + Y_i$$
> 
> مثال لو قوتنا $S = 2$ وعندنا تنينين الأول قوته 1 وبونص 99 والتاني قوته 100 وبونص 0 لو حاربنا التاني الأول هنموت لأن 2 أصغر من 100 لكن لو حاربنا الأول هنكسب لأن 2 أكبر من 1 وقوتنا هتبقى $2 + 99 = 101$
> 
> دلوقتي نقدر نحارب التاني لأن 101 أكبر من 100 ونكسب الجيم كله ونطبع `YES`

> [!TIP]
> 
> **Greedy Choice (الاختيار الجشع)**
> 
> ليه بنرتب بناء على قوة التنين بس ومبنبصش للـ `Bonus` بتاعه؟ لأن البونص مش هيفيدنا بحاجة لو إحنا أصلا مقدرناش نهزم التنين ومُتنا فالأولوية دايما للنجاة والنجاة بتيجي بمواجهة الأضعف استخدامك للـ `Struct` علشان تربط قوة التنين بالبونص بتاعه وتعملهم `Sort` مع بعض دي الخطوة المثالية جدا لحل المسألة دي

> [!IMPORTANT]
> 
> **الـ Complexity:**
> 
> - **Time Complexity:**
>   
>    بتاخد $O(N \log N)$ لأننا بنستخدم دالة `sort` لترتيب التنانين والـ `Loop` اللي بعدها بتاخد $O(N)$ بس الترتيب هو اللي بيسيطر على الوقت الكلي
> - **Space Complexity:**
>   
>    بتاخد $O(N)$ علشان بنخزن بيانات التنانين جوا مصفوفة الـ `Struct` 

> [!CAUTION]
> 
> **C++ Template:**
> ```cpp
> struct Dragon{
>   int x, y;
> };
> bool cmf(Dragon a, Dragon b){
>   return a.x < b.x;
> }
> void solve() {
>   int s, n; cin >> s >> n;
>   
>   vector<Dragon> drag(n);
>   for(int i = 0; i < n; i++) cin >> drag[i].x >> drag[i].y;
> 
>   sort(all(drag), cmf);
>   
>   for(int i = 0; i < n; i++){
>     if(s > drag[i].x) s += drag[i].y;
>     else {
>       cout << "NO" << endl;
>       return;
>     }
>   }
>   cout << "YES" << endl;
> }
> ```

## 13) Jzzhu and Children (جيزو والأطفال)

> [!NOTE]
>  **الفكرة التقنية (The Concept):**
>   المسألة بتطلب مننا نعرف مين أخر طفل هيروح بيته بعد ما كل الأطفال تاخد الكاندي بتاعتها بدل ما نعمل محاكاة `Simulation` ونستخدم `Queue` ونقعد نلف ونطرح ونرجع الطفل لأخر الطابور إحنا نقدر نحسبها بحسبة رياضية بسيطة جدا الفكرة كلها إننا بنحسب كل طفل محتاج يقف في الطابور كام مرة علشان ياخد كل الكاندي إللي هو عايزها الطفل إللي محتاج أكبر عدد من المرات هو دا إللي هيفضل للأخر ولو في كذا طفل محتاجين نفس أكبر عدد من المرات يبقى الطفل إللي ترتيبه متأخر في الطابور (الـ `Index` بتاعه أكبر) هو إللي هيروح أخر واحد علشان دوره بييجي متأخر

> [!TIP]
> 
> **Math Trick (الخدعة الرياضية)**
> 
> علشان نحسب عدد المرات بنقسم الكاندي إللي الطفل عايزها على الكاندي إللي بياخدها في المرة الواحدة $M$ وبنعمل تقريب للأكبر `ceil`
> 
> الطريقة بتاعتك ممتازة بإنك استخدمت `<= mx` علشان لو لقينا طفل محتاج نفس عدد المرات نحدث الإجابة وناخد الـ `Index` الجديد الأكبر

> [!IMPORTANT]
> 
> **القانون الرياضي والمثال:**
> 
> قانون حساب عدد اللفات لكل طفل
> 
> $$Turns = \lceil \frac{A_i}{M} \rceil$$
> 
> مثال لو الأطفال عايزين `[1, 3, 1, 4, 2]` وبنوزع $M = 2$
> 
> - الطفل الأول محتاج $\lceil 1/2 \rceil = 1$ لفة
>     
> - الطفل التاني محتاج $\lceil 3/2 \rceil = 2$ لفة
>     
> - الطفل التالت محتاج $\lceil 1/2 \rceil = 1$ لفة
>     
> - الطفل الرابع محتاج $\lceil 4/2 \rceil = 2$ لفة
>     
> - الطفل الخامس محتاج $\lceil 2/2 \rceil = 1$ لفة
>     
> 
> أكبر عدد لفات هو 2 وموجود عند الطفل التاني والرابع وبما إن الرابع ترتيبه ورا التاني إذن الرابع هو أخر واحد هيروح بيته

> [!WARNING]
> 
> **الـ Complexity:**
> 
> - **Time Complexity:**
>   
>    بتاخد $O(N)$ لأننا بنلف لفة واحدة بس على المصفوفة ونحسب المرات ونجيب الماكسيمم 
> - **Space Complexity:**
>   
>    بتاخد $O(N)$ علشان بنخزن احتياجات الأطفال جوا مصفوفة الـ `Vector`

> [!CAUTION]
> 
> **C++ Template:**
> ```cpp 
> void solve() {
>   int n, m; cin >> n >> m;
>   vi a(n); cin >> a;
> 
>   // بنحسب عدد المرات لكل طفل
>   for(int i = 0; i < n; i++) a[i] = ceil(a[i] / (double)m);
> 
>   int mx = 0, idx = 0;
>   for(int i = 0; i < n; i++){
>     // لو الطفل محتاج مرات أكبر من أو تساوي الماكسيمم بنحدث الإجابة
>     if(mx <= a[i]) mx = a[i], idx = i + 1;
>   }
>   cout << idx << endl;
> }
> ```

## 14) Make it Beautiful (خليها جميلة)

> [!NOTE]
>  **الفكرة التقنية (The Concept):**
>  
 المسألة بتطلب مننا نرتب المصفوفة علشان تكون `Beautiful` بمعنى إن مفيش ولا عنصر جوا المصفوفة يكون بيساوي مجموع العناصر إللي قبله بما إن المصفوفة جاية مترتبة تصاعديا وكل الأرقام موجبة يبقى دايما أكبر رقم موجود في الأخر وأصغر رقم موجود في الأول الخدعة هنا إننا لو حطينا أكبر رقم في أول المصفوفة خالص وبعده أصغر رقم كدا إحنا ضمنا إن المجموع التراكمي دايما هيكون أكبر من أي رقم هييجي بعد كدا لأننا بدأنا بأكبر قيمة موجودة أصلا الحالة الوحيدة إللي مستحيل نعالج فيها المصفوفة هي لو كانت كل الأرقام متساوية وساعتها أول عنصر هيكون بيساوي أخر عنصر بالظبط فبنطبع `NO` لأن أي ترتيب مش هيغير حاجة

> [!IMPORTANT]

>

## 15) The best 3seer 2sab (أحلى عصير قصب)

> [!NOTE]
> 
> **الفكرة التقنية (The Concept)**
> 
> المسألة دي تطبيق مباشر ومشهور جدا لخوارزمية الجشع `Greedy Algorithm` وتحديدا مسألة حقيبة الظهر الكسرية `Fractional Knapsack`
> 
> الفكرة ببساطة إننا عايزين نملى خزان حجمه $L$ بأغلى عصير قصب ممكن وبما إننا نقدر ناخد جزء من الكوباية ونسيب الباقي إذن أحسن استراتيجية هي إننا نحسب سعر اللتر الواحد لكل كوباية ونرتب الكوبايات تنازليا من الأغلى للأرخص.
> 
> بعد كدا بنمشي على الكوبايات المترتبة دي ولو الخزان فيه مساحة تاخد الكوباية كلها بناخدها ونطرح حجمها من مساحة الخزان ونزود سعرها على الإجمالي.
> 
> ولو الخزان مساحته بقت أصغر من حجم الكوباية بناخد الجزء إللي يكفي الخزان بس ونقفل على كدا ونطبع الناتج .

> [!IMPORTANT]
> 
> **القانون الرياضي والمثال**
> 
> قانون حساب سعر اللتر الواحد لكل كوباية
> 
> $$Ratio = \frac{V_i}{L_i}$$
> 
> وقانون حساب سعر الجزء المتبقي لو الخزان ميكفيش الكوباية كلها
> 
> $$Added Value = Remaining Space \times Ratio$$
> 
> مثال لو عندنا خزان حجمه 50 لتر وعندنا 3 كوبايات
> 
> 1. الكوباية الأولى 60 جنيه وحجمها 10 لتر إذن اللتر بـ 6 جنيه
>     
> 2. الكوباية التانية 100 جنيه وحجمها 20 لتر إذن اللتر بـ 5 جنيه
>     
> 3. الكوباية التالتة 120 جنيه وحجمها 30 لتر إذن اللتر بـ 4 جنيه
>     
> 
> خطوات الحل هتكون كدا
> 
> - [x] هناخد الأولى كلها علشان هي الأغلى الخزان هيفضل فيه 40 ومكسبنا 60
>     
> - [x] هناخد التانية كلها الخزان هيفضل فيه 20 ومكسبنا هيبقى $60 + 100 = 160$
>     
> - [x] التالتة الخزان محتاج 20 بس وهي 30 فهناخد 20 لتر بس سعرهم $20 \times 4 = 80$
>     
> - [x] الإجمالي النهائي هيبقى $160 + 80 = 240$
>     

> [!TIP]
> 
> **Precision Trick (خدعة الدقة العشرية)**
> 
> بما إننا بنحسب كسور وأرقام عشرية لازم ناخد بالنا من الدقة أو الـ `Precision` استخدام نوع `double` وقت حساب النسبة دي خطوة مهمة جدا علشان منتلغبطش في الكسور. وكمان وإحنا بنطبع الناتج لازم نستخدم دوال زي `fixed` مع `setprecision` علشان نطلع الناتج بنفس عدد الأصفار المطلوبة بالظبط.

> [!WARNING]
> 
> **الـ Complexity**
> 
> - **Time Complexity:**
>   
>    بتاخد $O(N \log N)$ علشان إحنا بنعمل `sort` للكوبايات بناء على سعر اللتر وحلقة الـ `Loop` بتاخد $O(N)$ فالترتيب هو إللي بياخد الوقت الأكبر.
> - **Space Complexity:**
>   
>    بتاخد $O(N)$ علشان بنخزن بيانات الكوبايات كلها جوا مصفوفة `vector` من نوع الـ `Struct` بتاعنا. 


> [!CAUTION]
> 
> **C++ Template**
> ```cpp
> struct Juice{
>   int Vi, Li;
> };
> 
> // دالة المقارنة علشان نرتب من الأغلى للأرخص للتر الواحد
> bool cmf(Juice a, Juice b){
>   return ((double)a.Vi / a.Li) > ((double)b.Vi / b. Li);
> }
> void solve() {
>   int N, L; cin >> N >> L;
> 
>   vector<Juice> j(N);
> 
>   for(int i = 0; i < N; i++) cin >> j[i].Vi >> j[i].Li;
> 
>   sort(all(j), cmf);
> 
>   double cnt{0.0};
>   for(int i = 0; i < N && L != 0; i++){ 
>     // لو الخزان يكفي الكوباية كلها
>     if(L >= j[i].Li) cnt += j[i].Vi, L -= j[i].Li;
> 
>     // لو الخزان مساحته متبقاش فيها كتير
>     else {
>       double rem = (double)j[i].Vi / j[i].Li;
>       cnt += L * rem, L = 0;
>     }
>   }
>   // طباعة الناتج بدقة 6 أرقام عشرية
>   cout << fixed << setprecision(6) << cnt << endl;
> }
> ```

## 16) Thorns and Coins (الأشواك والعملات)

> [!NOTE]
> 
> **الفكرة التقنية (The Concept)**
> 
> المسألة بتطلب مننا نجمع أكبر عدد من العملات المعدنية `@` واحنا بنتحرك في مسار معين مسموح لنا نتحرك خطوة واحدة أو خطوتين في المرة الواحدة الفكرة ببساطة إننا نقدر ننط من فوق شوكة واحدة `*` لأننا بنقدر ناخد خطوتين لكن لو قابلنا شوكتين ورا بعض `**` كدا الطريق بقى مسدود تماما ومستحيل نعدي لأن أقصى نطة لينا هي خطوتين بس علشان كدا إحنا هنمشي من أول المسار ونجمع أي عملة تقابلنا وأول ما نلاقي شوكتين متتاليين هنوقف اللف ونطبع إللي جمعناه لحد اللحظة دي

> [!IMPORTANT]
> 
> **القانون الرياضي والمثال**
> 
> شرط التوقف الإجباري بيكون كدا
> 
> $$S[i] == '*' \text{ and } S[i-1] == '*' \implies \text{Break}$$
> 
> مثال لو عندنا المسار دا `.@@*@.**@@`
> 
> - هنبدأ من الأول فاضي `.`
>     
> - هناخد العملة الأولى `@` الرصيد 1
>     
> - هناخد العملة التانية `@` الرصيد 2
>     
> - هننط من فوق الشوكة `*`
>     
> - هناخد العملة التالتة `@` الرصيد 3
>     
> - هنلاقي شوكتين ورا بعض `**` كدا الطريق وقف ومستحيل نوصل لأخر عملتين إذن أقصى عدد هو 3
>     

> [!TIP]
> 
> **Greedy Simulation (المحاكاة الجشعة)**
> 
> المسألة دي مش محتاجة لا برمجة ديناميكية `Dynamic Programming` ولا دالة `Recursion` معقدة كل إللي محتاجينه `Loop` واحدة بتلف على النص حرف حرف وبتعد العملات ولو شافت عائق مزدوج بتعمل `break` دا بيخلي الحل بسيط جدا وسريع

> [!WARNING]
> 
> **الـ Complexity**
> 
> - **Time Complexity:**
>   
>    بتاخد $O(N)$ لأننا بنلف على الـ `String` مرة واحدة بس وفي أسوأ الحالات هنوصل لآخره 
> - **Space Complexity:**
>   
>    بتاخد $O(N)$ علشان بنخزن المسار جوا متغير من نوع `String` 

> [!CAUTION]
> 
> **C++ Template**
> ```cpp
> void solve() {
>   int n; cin >> n;
>   string s; cin >> s;
> 
>   int cnt {0};
>   
>   for(int i = 0; i < n; i++) {
> 	  // لو لقينا عملة بنزود العداد
>     if(s[i] == '@') cnt++;
>     // لو لقينا شوكة والبطاقة إللي بعدها شوكة برده يبقى طريق مسدود
>     if(i + 1 < n && s[i] == '*' && s[i + 1] == '*')  break;
>   }
>   
>   cout << cnt << endl;
> }
> ```

## 17) Same Differences (الفروق المتشابهة)

> [!NOTE]
> 
> **الفكرة التقنية (The Concept)**
> 
> المسألة دي من أشهر المسائل إللي بتعتمد على تبسيط المعادلات الرياضية في البرمجة التنافسية الفكرة كلها إننا محتاجين نلاقي عدد الأزواج إللي بتحقق شرط معين وبدل ما نعمل حلقة تكرارية متداخلة `Nested Loops` وتجيب وقت طويل `Time Limit Exceeded` إحنا بنعيد ترتيب المعادلة علشان نخلي المتغيرات إللي ليها نفس الـ `Index` مع بعض في طرف واحد بمجرد ما نفصلهم المسألة بتتحول من إننا بندور على أزواج لإننا بندور على أرقام متكررة ودي بنقدر نحسبها بسهولة جدا باستخدام مصفوفة تكرار `Frequency Array`

> [!IMPORTANT]
> 
> **القانون الرياضي والمثال**
> 
> المعادلة الأصلية في المسألة بتيجي كدا
> 
> $$A_j - A_i = j - i$$
> 
> ولما ننقل المتغيرات علشان نجمع الـ $i$ في طرف والـ $j$ في طرف هتبقى كدا
> 
> $$A_j - j = A_i - i$$
> 
> مثال لو عندنا المصفوفة دي `[1, 6, 3, 4, 5, 6]` وبدأنا نرقم الـ `Index` من 1
> 
> - العنصر الأول قيمته 1 والاندكس بتاعه 1 إذن $1 - 1 = 0$
>     
> - العنصر التاني قيمته 6 والاندكس بتاعه 2 إذن $6 - 2 = 4$
>     
> - العنصر التالت قيمته 3 والاندكس بتاعه 3 إذن $3 - 3 = 0$
>     
> 
> هنلاقي إن الصفر اتكرر مرتين دا معناه إن العنصر الأول والتالت بيحققوا المعادلة إذن بنزود العداد بتاعنا

> [!TIP]
> 
> **Shift Trick (خدعة التشفيت للموجب)**
> 
> بما إننا بنطرح الـ `Index` من القيمة فالناتج ممكن جدا يطلع بالسالب وطبعا مستحيل نستخدم رقم سالب كـ `Index` جوا مصفوفة الـ `Frequency` بتاعتنا.
> 
> علشان كدا إحنا بنعمل خدعة رياضية ذكية وبنجمع قيمة $N$ على كل ناتج علشان نشفت الأرقام كلها وتكون موجبة دايما ونقدر نخزنها بأمان.

> [!WARNING]
> 
> **الـ Complexity**
> 
> - **Time Complexity:**
>   
>    بتاخد $O(N)$ علشان إحنا بنلف لفة واحدة بس على المصفوفة ونحسب القيمة اللحظية ونخزنها
> - **Space Complexity:**
>   
>    بتاخد $O(N)$ علشان بنخزن مصفوفة التكرار بحجم $2N + 5$ لاستيعاب كل الأرقام بعد التشفيت.

> [!CAUTION]
> 
> **C++ Template**
> ``` cpp
> void solve() {
>   int n; cin >> n;
>   vi a(n); cin >> a;
> 
>   // مصفوفة التكرار بحجم كافي لاستيعاب الأرقام بعد التشفيت
>   vi freq(2 * n + 5, 0);
>   int cnt {0};
>   for(int i = 0; i < n; i++){
>     // علشان ميبقاش سالب N مع تشفيت الرقم ب  a[i] - i  تطبيق المعادلة
>     a[i] = a[i] - i + n;
>     
>     // بنزود على العداد عدد المرات إللي شوفنا فيها الرقم دا قبل كدا
>     cnt += freq[a[i]];
>     // بنحدث مصفوفة التكرار
>     freq[a[i]]++;
>   }
>   cout << cnt << endl;
> }
> ```

## 18) Min Cost String (أقل تكلفة للنص)

> [!NOTE]
> 
> **الفكرة التقنية (The Concept)**
> 
> المسألة بتطلب مننا نبدل كل علامات الاستفهام في النص `String` بحروف علشان نوصل لأقل تكلفة ممكنة وكمان النص الناتج لازم يكون أصغر نص أبجديا لو في كذا إجابة صح الفكرة الجشعة `Greedy` هنا بتعتمد على نقطة ذكية جدا وهي إن تكلفة أي حرفين متتاليين شبه بعض دايما هتكون صفر لأننا بنطرح الرقم من نفسه.
> 
> دا معناه إن لو عندنا مجموعة من علامات الاستفهام ورا بعض أحسن حاجة نعملها إننا نخليهم كلهم حرف واحد بس متكرر .
> 
> كدا إحنا مش محتاجين نحسب غير التكلفة بين الحرف إللي هنختاره وبين الحرف إللي قبل البلوك بتاع علامات الاستفهام والحرف إللي بعده على طول.

> [!IMPORTANT]
> 
> **القانون الرياضي والمثال**
> 
> قانون حساب التكلفة للحرف المقترح $C$
> 
> $$Cost = |Cost(C) - Cost(Left)| + |Cost(C) - Cost(Right)|$$
> 
> مثال لو النص بتاعنا `a??c` وتكلفة `a` بـ 2 وتكلفة `c` بـ 5 وتكلفة `x` بـ 4
> 
> - هنعتبر إننا اخترنا الحرف `x` مكان علامات الاستفهام فالنص هيبقى `axxc`
>     
> - التكلفة بين `a` و `x` هي $|2 - 4| = 2$
>     
> - التكلفة بين `x` و `x` هي $|4 - 4| = 0$
>     
> - التكلفة بين `x` و `c` هي $|4 - 5| = 1$
>     
> - الإجمالي هيكون $2 + 0 + 1 = 3$
>     
>     إحنا بنعمل كدا مع كل الحروف من `a` لحد `z` ونختار الحرف إللي بيدينا أقل ناتج.
>     

> [!TIP]
> 
> **Lexicographical Trick (خدعة الترتيب الأبجدي)**
> 
> علشان نضمن إننا بنطبع أصغر نص أبجديا إحنا بنمشي بـ `Loop` على الحروف بالترتيب من `a` لحد `z` وبنعمل تحديث لقيمة الـ `Minimum` بس في حالة إننا لقينا تكلفة أقل تماما `<`.
> 
> لو لقينا حرف بيدينا نفس التكلفة مش بنعمل تحديث وبنحتفظ بالحرف الأولاني إللي لقيناه علشان هو الأصغر أبجديا اللوجيك دا بيخلص الشرطين في ضربة واحدة.

> [!WARNING]
> 
> **الـ Complexity**
> 
> - **Time Complexity:**
>   
>    بتاخد $O(N \times 26)$ يعني نقدر نعتبرها $O(N)$ لأننا بنمشي على النص مرة واحدة ولما بنلاقي علامات استفهام بنجرب 26 حرف بس فالموضوع سريع جدا.
> - **Space Complexity:**
>   
>    بتاخد $O(N)$ علشان بنخزن النص جوا متغير `String` وبنعدل عليه في الميموري

> [!CAUTION]
> 
> **C++ Template**
> ```cpp
> void solve() {
>   string s; cin >> s;
>   vi a(26); cin >> a;
> 
>   int n = s.length();
> 
>   for(int i = 0; i < n; i++) {
>     // لو لقينا علامة استفهام بنبدأ نشوف البلوك بتاعها
>     if(s[i] == '?'){ `[cite: 19]`
> 
>       int j = i;
>       while(j < n && s[j] == '?') j++;
> 
>       int mn = 2e9;
>       char bCh = 'a';
>       
>       // بنجرب كل الحروف علشان نلاقي الأقل في التكلفة
>       for(char c ='a'; c <= 'z'; c++){ 
>         int curcst {0};
> 
>         if(i > 0) curcst += abs(a[c - 'a'] - a[s[i - 1] - 'a']);
>         if(j < n) curcst += abs(a[c - 'a'] - a[s[j] - 'a']);
>         
>         if(curcst < mn) mn = curcst, bCh = c;
>       }
>       // بنملى البلوك كله بأحسن حرف لقيناه
>       for(int k = i; k < j; k++) s[k] = bCh;
>       i = j - 1;
>     }
>   }
>   
>   // حساب التكلفة الكلية وطباعتها
>   int tCst {0};
>   for (int i = 0; i < n - 1; i++) tCst += abs(a[s[i] - 'a'] - a[s[i + 1] - 'a']);
> 
>   cout << tCst << endl;
>   cout << s << endl;
> }
> ```

## 19) Missed Number (الرقم المفقود)

> [!NOTE]
> 
> **الفكرة التقنية (The Concept)**
> 
> المسألة بتطلب مننا نختار قيمة للرقم التالت $a_3$ علشان نحقق أكبر عدد ممكن من معادلات الفيبوناتشي إحنا عندنا 3 شروط أو 3 معادلات ممكن يتحققوا وكل معادلة فيهم بتدينا قيمة مفترضة للرقم التالت دا بنحسب التلات قيم دول ولو لقينا إن في قيمتين أو تلاتة بيساووا بعض دا معناه إننا نقدر نحقق أكتر من شرط في نفس الوقت لو اخترنا القيمة المشتركة دي

> [!IMPORTANT]
> 
> **القانون الرياضي والمثال**
> 
> علشان نحقق الشروط التلاتة عندنا 3 احتمالات لقيمة $a_3$
> 
> الاحتمال الأول
> 
> $$X = a_1 + a_2$$
> 
> الاحتمال التاني
> 
> $$Y = a_4 - a_2$$
> 
> الاحتمال التالت
> 
> $$Z = a_5 - a_4$$
> 
> بنقارن النتايج ببعض ولو مفيش أي حاجة بتساوي التانية يبقى هنقدر نحقق شرط واحد بس إما لو في اتنين بيساووا بعض هنحقق شرطين ولو التلاتة متساويين هنحقق 3 شروط
> 
> مثال لو المصفوفة `1 1 _ 3 5`
> 
> - $X = 1 + 1 = 2$
>     
> - $Y = 3 - 1 = 2$
>     
> - $Z = 5 - 3 = 2$
>     
> 
> التلات قيم طلعوا 2 إذن لو حطينا $a_3 = 2$ هنحقق التلات شروط ونطبع 3

> [!TIP]
> 
> **Logic Trick (خدعة التطابق)**
> 
> بدل ما نعمل شروط `if` كتير ونعقد الكود إحنا بنحسب كم مرة القيم دي بتساوي بعضها وبنجمعهم في متغير واحد.
> 
> لو مفيش تطابق الناتج بيكون 1 ولو في تطابق واحد الناتج بيكون 2 ولو التلاتة متطابقين المجموع هيكون 3 وبناخد المينيمم مع 3 علشان الناتج ميزيدش ودا لوجيك ذكي جدا بيختصر سطور كتير.

> [!WARNING]
> 
> **الـ Complexity**
> 
> - **Time Complexity:**
>   
>   بتاخد $O(1)$ لأننا بنعمل مجرد عمليات جمع وطرح ومقارنات بسيطة ومفيش أي `Loops`. 
> - **Space Complexity:**
>   
بتاخد $O(1)$ لأننا بنستخدم متغيرات عادية جدا ومش بنحجز أي مصفوفات.

> [!CAUTION]
> 
> **C++ Template**
> ```cpp
> void solve() {
>   int a1, a2, a4, a5; 
>   cin >> a1 >> a2 >> a4 >> a5;
> 
>   // بنحسب التلات احتمالات لقيمة الرقم المفقود
>   int x = a1 + a2, y = a4 - a2, z = a5 - a4;
> 
>   // بنشوف كام احتمال طلع متساوي مع التاني
>   int s = (x == y) + (y == z) + (x == z);
> 
>   // بنطبع الناتج ومستحيل يزيد عن 3
>   cout << min ((int)3, s + 1) << endl;
> }
> ```

## 20) Mewwwwwwwwo (مياووو)

> [!NOTE]
> 
> **الفكرة التقنية (The Concept)**
> 
> المسألة بسيطة جدا وبتطلب مننا نلاقي أطول مسافة فاضية `Blank Space` والمسافة دي عبارة عن أصفار متتالية ورا بعضها الفكرة كلها إننا بنمشي على الأرقام واحد واحد وبنعمل عداد `Counter` بيزيد لو قابلنا صفر ولو قابلنا واحد بنصفر العداد دا تاني وبنحتفظ دايما بأكبر قيمة وصلها العداد في متغير تاني

> [!IMPORTANT]
> 
> **القانون الرياضي والمثال**
> 
> معادلة تحديث العداد بتكون كدا
> 
> $$Counter = \begin{cases} Counter + 1 & \text{if } A_i = 0 \\ 0 & \text{if } A_i = 1 \end{cases}$$
> 
> وطبعا بنحدث الماكسيمم كدا
> 
> $$Max = \max(Max, Counter)$$
> 
> مثال لو عندنا المصفوفة دي `1 0 0 1 0`
> 
> - أول رقم 1 إذن العداد بصفر والماكسيمم بصفر
>     
> - تاني رقم 0 إذن العداد بقى 1 والماكسيمم بقى 1
>     
> - تالت رقم 0 إذن العداد بقى 2 والماكسيمم بقى 2
>     
> - رابع رقم 1 إذن العداد رجع صفر تاني بس الماكسيمم لسه 2
>     
> - خامس رقم 0 إذن العداد بقى 1 والماكسيمم لسه 2
>     
> 
> في الأخر بنطبع الماكسيمم إللي هو 2

> [!TIP]
> 
> **On-the-fly Processing (المعالجة اللحظية)**
> 
> بدل ما نحجز مصفوفة `Vector` بحجم $N$ ونخزن فيها الأرقام وبعدين نلف عليها إحنا ممكن نقرأ الرقم ونعالجه في نفس اللحظة زي ما إنت عامل في الكود بتاعك كدا بالظبط دا بيوفر مساحة في الميموري وبيخلي الكود أسرع بكتير

> [!WARNING]
> 
> **الـ Complexity**
> 
> - **Time Complexity:**
>   
>    بتاخد $O(N)$ لأننا بنقرا الأرقام مرة واحدة بس في `Loop` واحد. 
> - **Space Complexity:**
> 
> - بتاخد $O(1)$ لأننا بنستخدم متغيرات عادية لتخزين الأرقام والعداد ومش بنحجز أي مصفوفات. 

> [!CAUTION]
> 
> **C++ Template**
> ```cpp
> void solve() {
>   int n; cin >> n;
> 
>   int cnt {0}, mx{0};
>   while(n--){
>     int x; cin >> x;
>     // لو الرقم صفر بنزود العداد ولو واحد بنصفره
>     cnt = (x == 0 ? cnt + 1 : 0);
>     mx = max(cnt, mx);
>   }
>   cout << mx << endl;
> }
> ```

## 21) Pizza For Sameh (بيتزا لسامح)

> [!NOTE]
> 
> **الفكرة التقنية (The Concept)**
> 
> المسألة دي عبارة عن مسألة رياضيات بحتة، بتطلب مننا نلاقي كل قيم $X$ إللي بتخلي الدالة $f(x) = x$. الفكرة كلها إننا بناخد المعادلة ونساويها بـ $X$ ونحلها كمعادلة جبرية عادية جدا بالورقة والقلم. لما نبسط المعادلة هنلاقي إن دايما في 3 حلول ثابتين بيعتمدوا على قيمة $N$ المكتوبة في المدخلات بس.

> [!IMPORTANT]
> 
> **القانون الرياضي والمثال**
> 
> المعادلة الأصلية في المسألة بتيجي كدا:
> 
> $$x = \frac{5xN^2}{x^2 + 3xN - 5N^2}$$
> 
> الحل الأول البديهي هو إننا نحط $X = 0$ وساعتها الطرفين هيساووا صفر إذن $0$ دا أول حل. لو افترضنا إن $X$ مش بصفر، نقدر نقسم الطرفين على $X$ وهتبقى المعادلة كدا:
> 
> $$1 = \frac{5N^2}{x^2 + 3xN - 5N^2}$$
> 
> نعمل طرفين في وسطين:
> 
> $$x^2 + 3xN - 5N^2 = 5N^2$$
> 
> نودي الـ $5N^2$ الناحية التانية علشان نصفر المعادلة:
> 
> $$x^2 + 3xN - 10N^2 = 0$$
> 
> نحللها لقوسين زي معادلات الدرجة التانية بالظبط:
> 
> $$(x + 5N)(x - 2N) = 0$$
> 
> إذن القيمتين التانيين هما $-5N$ و $2N$.
> 
> الخلاصة إن دايما عدد الحلول هو 3 وقيمهم هي $0$ و $-5N$ و $2N$.
> 
> مثال لو المدخل $N = 3$:
> 
> - الحل الأول: $0$
>     
> - الحل التاني: $-5 \times 3 = -15$
>     
> - الحل التالت: $2 \times 3 = 6$
>     

> [!TIP]
> 
> **Math Simplification (التبسيط الرياضي)**
> 
> في المسائل إللي زي دي، البرمجة وتجربة الأرقام بـ `Loop` مش هتفيدنا وهتجيب `Time Limit Exceeded`. بمجرد ما تحل المعادلة على الورق، الكود بيتحول لسطر طباعة واحد بيطلع الحلول الثابتة، ودا بيوفر وقت تفكير ومجهود كبير جدا في المسابقة.

> [!WARNING]
> 
> **الـ Complexity**
> 
> - **Time Complexity:**
>   
>    بتاخد $O(1)$ لأننا بنعمل 3 عمليات ضرب بسيطة جدا وبنطبع الناتج على طول في نفس اللحظة. 
> - **Space Complexity:**
>   
>    بتاخد $O(1)$ لأننا مش بنستخدم أي مصفوفات أو مساحات زيادة في الميموري. 

> [!CAUTION]
> 
> **C++ Template**
> ```cpp
> void solve() {
>   int N; 
>   cin >> N;
> 
>   // طباعة عدد الحلول وبعدين التلات حلول المستنتجة
>   cout << 3 << endl << -5 * N << ' ' << 0 << ' ' << 2 * N << endl;
> }
> ```

## 22) My Last Sorting Problem (مشكلة الترتيب الأخيرة)

> [!NOTE] 
> **الفكرة التقنية (The Concept)**
>  المسألة بتطلب مننا نرتب مجموعة من الكتب بناء على أسماءها بس بطريقة دمج بين الترتيب التصاعدي والتنازلي الفكرة ببساطة إننا بنمشي على حروف الكلمتين حرف حرف وأول ما نلاقي اختلاف بنشوف مكان الحرف دا فين بالظبط لو كان الحرف في مكان فردي (يعني الاندكس بتاعه زوجي `0, 2, 4` في البرمجة) بنرتب تصاعديا من `A` للـ `Z` ولو كان الحرف في مكان زوجي (يعني الاندكس بتاعه فردي `1, 3, 5` في البرمجة) بنرتب تنازليا من `Z` للـ `A` بمجرد ما نبني دالة المقارنة الخاصة بينا `Custom Comparator` نقدر نبعتها لدالة `sort` الجاهزة وهي هتعمل السحر كله وترتبهم

> [!IMPORTANT]
> 
> **القانون الرياضي والمثال**
> 
> شرط الترتيب جوا دالة المقارنة بيكون كدا بناء على الاندكس
> 
> $$Index \pmod 2 == 0 \implies A[i] < B[i]$$
> 
> $$Index \pmod 2 \neq 0 \implies A[i] > B[i]$$
> 
> مثال لو بنقارن الكلمتين `AB` و `AZ`
> 
> - في الاندكس `0` (مكان فردي) الحرفين `A` و `A` متساويين إذن نكمل
>     
> - في الاندكس `1` (مكان زوجي) الحرفين `B` و `Z` مختلفين
>     
> 
> بما إننا في مكان زوجي إذن الترتيب تنازلي يعني الحرف الأكبر أبجديا بييجي الأول وبما إن `Z` أكبر من `B` إذن الكلمة `AZ` هتيجي قبل `AB` في الترتيب النهائي

> [!TIP] 
> **Custom Sorting Trick (خدعة الترتيب المخصص)**
>  المسألة مش عايزانا نطبع الكلمات مترتبة دي عايزانا نطبع أرقامها الأصلية (الاندكس بتاعها قبل الترتيب) علشان كدا استخدامك للـ `Struct` علشان تربط الكلمة بالرقم بتاعها دي أفضل طريقة علشان الرقم يفضل مربوط بالكلمة حتى بعد ما أماكنهم تتغير جوا المصفوفة

> [!WARNING]
> 
> **الـ Complexity**
> 
> - **Time Complexity:**
>   
>    بتاخد $O(N \log N \times M)$ علشان إحنا بنعمل ترتيب لـ $N$ كلمة وكل مقارنة بين كلمتين ممكن تاخد وقت لحد $M$ في أسوأ الحالات بس علشان الـ `break` الضمني في الكود بتخلص أسرع بكتير 
> - **Space Complexity:**
>   
>    بتاخد $O(N \times M)$ علشان بنخزن الكلمات جوا مصفوفة الـ `Vector` بتاعنا 

> [!CAUTION] 
> **C++ Template**
> ```cpp
> struct Book{
>   string ttl;
>   int idx;
> };
> 
> // دالة المقارنة المخصصة للترتيب التصاعدي والتنازلي
> bool cmf(const Book& a, const Book& b){
>   for(int i = 0; i < (int)a.ttl.size(); i++){
>     if(a.ttl[i] != b.ttl[i]) 
>       // لو الاندكس زوجي بنرتب تصاعدي ولو فردي بنرتب تنازلي
>       return (i % 2 == 0) ? (a.ttl[i] < b.ttl[i]) : (a.ttl[i] > b.ttl[i]);
>   }
>   return false;
> }
> 
> void solve() {
>   int n, m; cin >> n >> m;
>   vector<Book> arr(n);
> 
>   for(int i = 0; i < n; i++) {
>     cin >> arr[i].ttl;
>     // حفظ الاندكس الأصلي علشان الطباعة
>     arr[i].idx = i + 1;
>   }
>   // تطبيق الترتيب
>   sort(all(arr), cmf);
>   // طباعة الاندكس الأصلي للكلمات بعد الترتيب
>   for(int i = 0; i < n; i++) cout << arr[i].idx << ' ';
>   cout << endl;
> 
> }
> ```
