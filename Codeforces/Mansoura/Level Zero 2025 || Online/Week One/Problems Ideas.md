## 1) Area of a Circle (مساحة الدائرة)

> [!NOTE]
> 
> **الفكرة التقنية:**
> 
> علشان نحسب مساحة الدائرة بنعوض تعويض مباشر في القانون الرياضي الثابت ويفضل نستخدم دالة `acos(-1)` علشان نجيب قيمة `pi` بدقة عالية جدا بدل ما نكتبها بإيدينا
> 
> **القانون الرياضي والمثال:**
> 
> $$A = \pi \times R^2$$
> 
> مثال: لو عندنا نصف القطر 5 المساحة هتكون $\pi$ مضروبة في 25 يعني النتيجة هتطلع تقريباً 78.539

> [!CAUTION]
> 
> **`C++` Template:**
> 
> C++
> 
> ```
> double R;
> cin >> R;
> double pi = acos(-1);
> double area = pi * R * R;
> cout << fixed << setprecision(9) << area << "\n";
> ```

## 2) Sum of the Two Largest (مجموع أكبر رقمين من تلاتة)

> [!NOTE]
> 
> **الفكرة التقنية:**
> 
> علشان نجيب مجموع أكبر رقمين بين 3 أرقام بنستخدم فكرة عكسية ذكية جدا بنجمع التلات أرقام كلهم ونطرح منهم أصغر رقم دا بيوفر علينا نكتب شروط `if` معقدة جوا الكود
> 
> **القانون الرياضي والمثال:**
> 
> $$MaxSum = (A + B + C) - \min(A, B, C)$$
> 
> مثال: لو الأرقام 5 و 10 و 3 المجموع الكلي 18 نطرح منه أصغر رقم إللي هو 3 يتبقى 15 ودا فعلا مجموع الـ 10 والـ 5

> [!TIP]
> 
> **`CP Pro Tip: min({})`**
> 
> في لغة `C++` نقدر نبعت أكتر من رقمين لدالة `min` أو `max` عن طريق إننا نحطهم جوا أقواس مجموعة `{}` إللي هي `Initializer List` كدا الكود بيبقى أقصر وأنضف

> [!CAUTION]
> 
> **`C++` Template:**
> 
> C++
> 
> ```
> int A, B, C;
> cin >> A >> B >> C;
> int max_sum = (A + B + C) - min({A, B, C});
> cout << max_sum << "\n";
> ```

## 3) Even and Odd Check (فحص الرقم الزوجي والفردي)

> [!NOTE]
> 
> **الفكرة التقنية:**
> 
> بنختبر الرقم هل هو زوجي ولا فردي عن طريق باقي قسمته على 2 لو الباقي صفر يبقى زوجي ولو مش صفر يبقى فردي
> 
> **القانون الرياضي والمثال:**
> 
> للرقم الزوجي $N \pmod 2 == 0$
> 
> للرقم الفردي $N \pmod 2 \neq 0$
> 
> مثال: لو الرقم 4 باقي قسمته على 2 بصفر يبقى زوجي ولو الرقم 5 باقي قسمته 1 يبقى فردي

> [!TIP]
> 
> **`CP Pro Tip: Bitwise AND (& 1)`**
> 
> المحترفين بيستخدموا عمليات الـ `Bitwise` علشان أسرع بكتير من باقي القسمة عملية `w & 1` بتفحص أول `Bit` في الرقم لو كان صفر يبقى الرقم زوجي ولو واحد يبقى فردي

> [!CAUTION]
> 
> **`C++` Template:**
> 
> C++
> 
> ```
> int w;
> cin >> w;
> 
> // الطريقة التقليدية
> if (w % 2 == 0) cout << "Even" << "\n";
> 
> // الطريقة الأسرع والأشيك
> if ((w & 1) == 0) cout << "Even" << "\n";
> ```

## 4) Extract Last Digit (استخراج الخانة الأخيرة)

> [!NOTE]
> 
> **الفكرة التقنية:**
> 
> علشان نجيب آخر خانة يعني الآحاد من أي رقم بنجيب باقي قسمته على 10
> 
> **القانون الرياضي والمثال:**
> 
> $$LastDigit = N \pmod{10}$$
> 
> مثال: لو الرقم 1234 عملنا باقي قسمة على 10 النتيجة هتكون 4

> [!WARNING]
> 
> **`CP Edge Case: Negative Numbers`**
> 
> لو الرقم سالب مثلا -123 عملية `-123 % 10` هترجع -3 بالسالب علشان نتجنب الـ `Bugs` دي دايما بنستخدم دالة القيمة المطلقة `abs(N % 10)`

> [!TIP]
> 
> **`CP Pro Tip: Huge Numbers`**
> 
> لو الرقم عملاق ومش هينفع يتخزن جوا `long long` بنقراه كـ `string` وناخد آخر حرف ونحوله لرقم عن طريق طرح قيمة الصفر في جدول الـ `ASCII` كدا `s.back() - '0'`

> [!CAUTION]
> 
> **`C++` Template:**
> 
> C++
> 
> ```
> int N;
> cin >> N;
> 
> // الطريقة الآمنة للأرقام الصحيحة
> int last_digit = abs(N % 10);
> cout << "Last digit is: " << last_digit << "\n";
> ```

## 5) Intersection of Two Intervals (تقاطع فترتين)

> [!NOTE]
> 
> **الفكرة التقنية:**
> 
> علشان نحسب فترة التقاطع بين أي فترتين بنجيب أكبر رقم في البدايات وأصغر رقم في النهايات
> 
> **القانون الرياضي والمثال:**
> 
> بداية التقاطع `max(L1, L2)` ونهاية التقاطع `min(R1, R2)`
> 
> مثال: الفترة الأولى من 1 لـ 5 والتانية من 3 لـ 7 أكبر بداية هي 3 وأصغر نهاية هي 5 إذن التقاطع من 3 لـ 5

> [!IMPORTANT]
> 
> **شرط التقاطع الحاسم**
> 
> التقاطع بيكون موجود فقط لو كانت البداية الجديدة أصغر من أو تساوي النهاية الجديدة لو العكس يبقى الفترتين مش متقاطعين أصلا

> [!CAUTION]
> 
> **`C++` Template:**
> 
> C++
> 
> ```
> int start = max(L1, L2);
> int end = min(R1, R2);
> 
> if (start <= end) cout << start << " " << end << "\n";
> else cout << -1 << "\n";
> ```

## 6) Triangle Inequality (متباينة المثلث)

> [!NOTE]
> 
> **الفكرة التقنية:**
> 
> علشان نتأكد هل الـ 3 أضلاع ينفعوا يعملوا مثلث ولا لأ لازم يكون مجموع أصغر ضلعين أكبر تماما من الضلع الأطول
> 
> **القانون الرياضي والمثال:**
> 
> $$A + B > C$$
> 
> بافتراض إن C هو الأطول مثال: لو الأضلاع 3 و 4 و 5 مجموع 3 و 4 يساوي 7 والـ 7 أكبر من الـ 5 إذن ينفع يكونوا مثلث

> [!TIP]
> 
> **`CP Pro Trick: Calculating the Gap`**
> 
> لو الأضلاع مش بتعمل مثلث وعايزين نعرف أقل قيمة نزودها على ضلع صغير علشان نقفل المثلث بنحسبها كدا $Increment = MaxSide - SumOfOtherTwo + 1$

> [!CAUTION]
> 
> **`C++` Template:**
> 
> C++
> 
> ```
> int max_side = max({a, b, c});
> int sum_others = (a + b + c) - max_side;
> 
> if (sum_others > max_side) cout << 0 << "\n"; 
> else cout << (max_side - sum_others) + 1 << "\n"; 
> ```

## 7) Summation from 1 to N (مجموع الأعداد من 1 لـ N)

> [!NOTE]
> 
> **الفكرة التقنية:**
> 
> علشان نحسب مجموع الأعداد المتتالية من 1 لحد رقم معين بنستخدم قانون رياضي مباشر بدل ما نلف بـ `Loop` وناخد وقت طويل
> 
> **القانون الرياضي والمثال:**
> 
> $$Sum = \frac{N \times (N + 1)}{2}$$
> 
> مثال: مجموع الأرقام من 1 لـ 100 هيكون 100 مضروبة في 101 مقسومة على 2 النتيجة 5050

> [!WARNING]
> 
> **`CP Trap: Time Limit & Overflow`**
> 
> لو الرقم كبير جدا استخدام الـ `Loop` هيجيب `Time Limit Exceeded` وناتج الضرب ممكن يعمل `Overflow` فلازم دايما نستخدم `long long`

> [!CAUTION]
> 
> **`C++` Template:**
> 
> C++
> 
> ```
> long long n;
> cin >> n;
> 
> // Time Complexity O(1)
> long long total_sum = (n * (n + 1)) / 2;
> cout << total_sum << "\n";
> ```

## 9) Sum of Two Equals the Third (رقم يساوي مجموع الآخرين)

> [!NOTE]
> 
> **الفكرة التقنية:**
> 
> علشان نختبر هل في رقم جوا تلات أرقام بيساوي مجموع الاتنين التانيين بنجيب الرقم الأكبر ونطرحه من المجموع الكلي ونتأكد هل الباقي بيساويه ولا لأ
> 
> **القانون الرياضي والمثال:**
> 
> نختبر هل $Largest == (A + B + C) - Largest$
> 
> مثال: لو الأرقام 2 و 5 و 3 الأكبر هو 5 المجموع الكلي 10 نطرح منه 5 يتبقى 5 إذن الـ 5 بتساوي مجموع التانيين

> [!TIP]
> 
> **`CP Pro Tip: The Max-Sum Logic`**
> 
> الخدعة دي بتوفر علينا نكتب تباديل كتير جوا شروط الـ `if` وتخلي اللوجيك بسيط وسريع

> [!CAUTION]
> 
> **`C++` Template:**
> 
> C++
> 
> ```
> int a, b, c;
> cin >> a >> b >> c;
> 
> int largest = max({a, b, c});
> int sum_others = (a + b + c) - largest;
> 
> if (largest == sum_others) cout << "YES\n";
> else cout << "NO\n";
> ```

## 10) Average 2 (المتوسط الموزون)

> [!NOTE]
> 
> **الفكرة التقنية:**
> 
> بنحسب المتوسط الموزون لما يكون كل قيمة ليها وزن مختلف بنضرب كل قيمة في وزنها ونجمعهم ونقسم على مجموع الأوزان كلها
> 
> **القانون الرياضي والمثال:**
> 
> $$MEDIA = \frac{\sum (x_i \cdot w_i)}{\sum w_i}$$
> 
> مثال: لو الدرجات 5 بوزن 2 و 10 بوزن 3 هنضرب 5 في 2 بـ 10 و 10 في 3 بـ 30 المجموع 40 نقسمه على مجموع الأوزان إللي هو 5 النتيجة هتكون 8

> [!TIP]
> 
> **`CP Pro Tip: Weighted Average Logic`**
> 
> إحنا مش بنقسم على عدد المواد إحنا بنقسم على مجموع الأوزان ولازم نستخدم `double` ونظبط الدقة بـ `setprecision(1)` علشان النتيجة تطلع مظبوطة

> [!CAUTION]
> 
> **`C++` Template:**
> 
> C++
> 
> ```
> double A, B, C;
> cin >> A >> B >> C;
> 
> double media = ((A * 2) + (B * 3) + (C * 5)) / 10.0;
> cout << "MEDIA = " << fixed << setprecision(1) << media << "\n";
> ```

## 11) Calculating Function (دالة المجموع المتذبذب)

> [!NOTE]
> 
> **الفكرة التقنية:**
> 
> لحساب ناتج متسلسلة بتغير إشارتها بين سالب وموجب بنستنتج نمط رياضي بدل ما نجمعهم واحد واحد
> 
> **القانون الرياضي والمثال:**
> 
> لو الرقم زوجي النتيجة هتكون $N / 2$
> 
> لو الرقم فردي النتيجة هتكون $-(N + 1) / 2$
> 
> مثال: لو الرقم 4 النتيجة هتكون 2 ولو الرقم 5 النتيجة هتكون -3

> [!TIP]
> 
> **`CP Pro Tip: Avoid O(N) Loops`**
> 
> الرقم ممكن يوصل لـ $10^{15}$ فالـ `Loop` هتجيب `TLE` علطول الحل هو النمط الرياضي الثابت دا مع استخدام `long long`

> [!CAUTION]
> 
> **`C++` Template:**
> 
> C++
> 
> ```
> long long n;
> cin >> n;
> 
> if (n % 2 == 0) cout << n / 2 << "\n";
> else cout << -(n + 1) / 2 << "\n";
> ```

## 12) Cyclic Consumption (الاستهلاك الدوري)

> [!NOTE]
> 
> **الفكرة التقنية:**
> 
> علشان نعرف مين إللي هيخلص المورد في دورة بتتكرر بنختصر كل الدورات الكاملة باستخدام باقي القسمة ونشتغل على الباقي بس
> 
> **القانون الرياضي والمثال:**
> 
> $$V_{rem} = V \pmod{A + B + C}$$
> 
> مثال: لو المورد 10 والاستهلاك 2 و 3 و 4 المجموع 9 باقي قسمة 10 على 9 هو 1 الشخص الأول محتاج 2 والمتبقي 1 إذن هو إللي المورد هيخلص معاه

> [!TIP]
> 
> **`CP Pro Tip: The Modulo Shortcut`**
> 
> استخدام باقي القسمة بيخلي الحل في زمن $O(1)$ ودا بيوفر محاكاة لآلاف الدورات إللي ملهاش لازمة

> [!CAUTION]
> 
> **`C++` Template:**
> 
> C++
> 
> ```
> V %= (A + B + C);
> 
> if (V < A) cout << "F\n";
> else if (V < (A + B)) cout << "M\n";
> else cout << "T\n";
> ```

## 13) Tricky Sum (المجموع المشروط بقوى العدد 2)

> [!NOTE]
> 
> **الفكرة التقنية:**
> 
> بنجمع كل الأرقام ونطرح منها ضعف الأرقام إللي تعتبر من قوى العدد 2 علشان نعكس إشارتها من موجب لسالب
> 
> **القانون الرياضي والمثال:**
> 
> $TotalSum = \frac{n \times (n + 1)}{2}$
> 
> $Result = TotalSum - (2 \times SumPowersOfTwo)$
> 
> مثال: لو الرقم 4 المجموع الكلي 10 قوى العدد 2 هي 1 و 2 و 4 مجموعهم 7 نضربهم في 2 يبقى 14 نطرحهم من 10 النتيجة -4

> [!TIP]
> 
> **`CP Pro Tip: Binary Growth Speed`**
> 
> قوى العدد 2 بتكبر بسرعة جدا يعني هنلف حوالي 30 لفة بس لغاية المليار فالكود هيكون سريع جدا

> [!CAUTION]
> 
> **`C++` Template:**
> 
> C++
> 
> ```
> long long n;
> cin >> n;
> 
> long long total_sum = n * (n + 1) / 2;
> long long sum_powers = 0;
> for (long long i = 1; i <= n; i *= 2) sum_powers += i;
> 
> long long result = total_sum - (2 * sum_powers);
> cout << result << "\n";
> ```

## 14) Buy a Shovel (شراء الجاروف بدون فكة)

> 
> **`CP Pro Tip: The Limit of 10`**

> لأننا شغالين على آحاد الأرقام الأنماط بتتكرر كل 10 مرات فمستحيل نحتاج نلف أكتر من 10 لفات جوا الـ `Loop`

> 
> C++
> 
> ```

>

## 16) Sorting Three Numbers (ترتيب ثلاثة أرقام)

> [!NOTE]
> 
> **الفكرة التقنية:**
> 
> علشان نرتب تلات أرقام من غير دوال الترتيب الجاهزة بنجيب أكبر وأصغر رقم ونطرحهم من المجموع الكلي علشان نجيب الرقم إللي في النص
> 
> **القانون الرياضي والمثال:**
> 
> $Middle = (A + B + C) - (Smallest + Largest)$
> 
> مثال: لو الأرقام 8 و 2 و 5 الأصغر 2 والأكبر 8 المجموع الكلي 15 نطرح منه 10 يتبقى 5 ودا الرقم إللي في النص

> [!TIP]
> 
> **`CP Pro Tip: The Sum Elimination Trick`**
> 
> الخدعة دي بتوفر علينا كتابة احتمالات كتير جوا شروط الـ `if` وبتخلي الكود نضيف جدا وسريع جدا لترتيب تلات عناصر بس

> [!CAUTION]
> 
> **`C++` Template:**
> 
> C++
> 
> ```
> long long a, b, c;
> cin >> a >> b >> c;
> 
> long long smallest = min({a, b, c});
> long long largest = max({a, b, c});
> long long middle = (a + b + c) - (smallest + largest);
> 
> cout << smallest << " " << middle << " " << largest << "\n";
> ```

## 17) Sum of k Distinct Odds (مجموع k أعداد فردية مختلفة)

> [!NOTE]
> 
> **الفكرة التقنية:**
> 
> علشان نقدر نكون رقم من عدد معين من الأرقام الفردية المختلفة لازم المجموع يكون معدي أو بيساوي مربع العدد دا ولازم يكونوا من نفس النوع فردي أو زوجي
> 
> **القانون الرياضي والمثال:**
> 
> المجموع الأدنى $N \ge K^2$ وشرط الزوجية $N \pmod 2 == K \pmod 2$
> 
> مثال: لو عايزين نكون 10 من 2 أرقام فردية مختلفة مربع الـ 2 هو 4 والـ 10 أكبر من الـ 4 والاتنين أرقام زوجية إذن ينفع زي 3 و 7

> [!TIP]
> 
> **`CP Pro Tip: The Square of Odds`**
> 
> القاعدة الرياضية بتقول إن مجموع أول $k$ أعداد فردية دايما بيساوي $k^2$ ودي بتختصر المسألة في معادلة واحدة من غير أي محاكاة

> [!CAUTION]
> 
> **`C++` Template:**
> 
> C++
> 
> ```
> long long n, k;
> cin >> n >> k;
> 
> if (n >= k * k && (n % 2 == k % 2)) cout << "YES\n";
> else cout << "NO\n";
> ```

## 18) Race Ranking (عدّ المتسابقين في المقدمة)

> [!NOTE]
> 
> **الفكرة التقنية:**
> 
> بنجمع نواتج الشروط المنطقية علطول كأنها أرقام علشان نعد الكام واحد سبقونا بدل ما نكتب شروط وتفرعات كتير
> 
> **القانون الرياضي والمثال:**
> 
> $$Count = [B > A] + [C > A] + [D > A]$$
> 
> مثال: لو المسافات 5 للأساسي والتانيين 8 و 2 و 7 الشرط هيطلع 1 و 0 و 1 مجموعهم 2 يعني في اتنين سبقوه

> [!TIP]
> 
> **`CP Pro Tip: Boolean Arithmetic`**
> 
> التكنيك دا اسمه `Branchless Programming` بيخلي الكود مفيهوش تفرعات وبيخلي تنفيذه أسرع على المعالج لأن ناتج الشرط بيتعامل كـ 1 أو 0

> [!CAUTION]
> 
> **`C++` Template:**
> 
> C++
> 
> ```
> int a, b, c, d;
> cin >> a >> b >> c >> d;
> 
> int result = (a < b) + (a < c) + (a < d);
> cout << result << "\n";
> ```

## 19) Summation Identities (الفرق بين مربع المجموع ومجموع المربعات)

> [!NOTE]
> 
> **الفكرة التقنية:**
> 
> بنحسب مربع المجموع ومجموع المربعات باستخدام قوانين المتسلسلات الرياضية ونطرحهم من بعض علشان نوفر وقت التنفيذ
> 
> **القانون الرياضي والمثال:**
> 
> مربع المجموع $S_2 = (\frac{n(n+1)}{2})^2$
> 
> مجموع المربعات $S_1 = \frac{n(n+1)(2n+1)}{6}$
> 
> مثال: لو الرقم 3 المجموع 6 مربعه 36 ومجموع المربعات 1 زائد 4 زائد 9 بيساوي 14 نطرحهم النتيجة 22

> [!TIP]
> 
> **`CP Pro Tip: The 0.5s Time Constraint`**
> 
> استخدام القوانين دي بيخلي الحل يشتغل في $O(1)$ ودا إللي هيخليك تعدي من حاجز الوقت الضيق للمسألة دي

> [!CAUTION]
> 
> **`C++` Template:**
> 
> C++
> 
> ```
> long long n;
> cin >> n;
> 
> long long sum = (n * (n + 1)) / 2;
> long long s2 = sum * sum;
> long long s1 = (n * (n + 1) * (2 * n + 1)) / 6;
> 
> cout << s2 - s1 << "\n";
> ```

## 20) Distance to Origin (المسافة إلى نقطة الأصل)

> [!NOTE]
> 
> **الفكرة التقنية:**
> 
> بنقارن بعد النقط عن المركز عن طريق حساب مربع المسافة علشان نهرب من استخدام الجذور التربيعية ونحافظ على الدقة
> 
> **القانون الرياضي والمثال:**
> 
> بنحسب المربع كدا $D^2 = X^2 + Y^2$
> 
> مثال: النقطة الأولى (3, 4) مربع مسافتها 25 والتانية (1, 1) مربع مسافتها 2 إذن التانية هي الأقرب

> [!TIP]
> 
> **`CP Pro Tip: Avoid Square Roots`**
> 
> دايما ابعد عن دالة `sqrt` في مسائل المقارنات الهندسية لأنها بتعمل أخطاء في الدقة العشرية وممكن تجيبلك `Wrong Answer`

> [!CAUTION]
> 
> **`C++` Template:**
> 
> C++
> 
> ```
> long long xr, yr, xw, yw;
> cin >> xr >> yr >> xw >> yw;
> 
> long long russo_dist = (xr * xr) + (yr * yr);
> long long wil_dist = (xw * xw) + (yw * yw);
> 
> if (russo_dist < wil_dist) cout << "Russo\n";
> else if (wil_dist < russo_dist) cout << "Wil\n";
> else cout << "Empate\n";
> ```

## 21) Range Sum (مجموع الأرقام في نطاق محدد)

> [!NOTE]
> 
> **الفكرة التقنية:**
> 
> بنحسب مجموع الأرقام في فترة معينة باستخدام قانون المتسلسلة الحسابية بدل ما نلف عليهم واحد واحد
> 
> **القانون الرياضي والمثال:**
> 
> عدد الأرقام $n = R - L + 1$ والمجموع $Sum = \frac{n \times (L + R)}{2}$
> 
> مثال: الفترة من 3 لـ 5 عددهم 3 المجموع هيكون 3 مضروبة في 8 مقسومة على 2 النتيجة 12

> [!TIP]
> 
> **`CP Pro Tip: Big Numbers & O(1)`**
> 
> القانون دا بيحل المسألة في خطوة واحدة مهما كانت الأرقام كبيرة بس لازم دايما نتأكد إننا شغالين بـ `long long` علشان الـ `Overflow`

> [!CAUTION]
> 
> **`C++` Template:**
> 
> C++
> 
> ```
> long long L, R;
> cin >> L >> R;
> 
> long long n = R - L + 1;
> long long sum = (n * (L + R)) / 2;
> 
> cout << sum << "\n";
> ```
