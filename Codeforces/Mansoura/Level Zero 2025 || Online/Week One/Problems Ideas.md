> ## 2) Sum of the Two Largest (مجموع أكبر رقمين من تلاتة)
> لإيجاد مجموع أكبر رقمين بين 3 أرقام (أو استبعاد أصغر رقم) بنستخدم القاعدة العكسية بدلاً من الشروط المعقدة:
> 
> **القانون الرياضي:**
> `MaxSum = (A + B + C) - min(A, B, C)`

>

> [!TIP] 
> **CP Pro Tip**
> استخدم `min({a, b, c})`
>

>

> [!CAUTION]
> **C++ Template:**
> ```cpp
> int A, B, C;
> cin >> A >> B >> C;
> int max_sum = (A + B + C) - min({A, B, C});
> cout << max_sum << "\n";
> ```

> ## 3) Even and Odd Check (فحص الرقم الزوجي والفردي)
> الطريقة الرياضية والبرمجية الأساسية لمعرفة ما إذا كان الرقم زوجياً أم فردياً هي التحقق من باقي قسمته على 2 (`% 2`).
> 
> - **الرقم الزوجي (Even):** `w % 2 == 0`
> - **الرقم الفردي (Odd):** `w % 2 != 0`
> 

> [!TIP]
> **CP Pro Tip: Bitwise AND (`& 1`) ⚡**
>
> في عالم الـ Competitive Programming، المحترفين بيستخدموا عمليات البت (Bitwise Operations) لأنها أسرع بكثير من باقي القسمة. عملية `w & 1` بتفحص أول بت (Bit) في الرقم، لو كان صفر يبقى الرقم زوجي، ولو واحد يبقى فردي.
> - **للزوجي:** `(w & 1)  == 0`
> - **للفردي:** `(w & 1)`
> 

> [!CAUTION]
**C++ Template:**
> ```cpp
> int w;
> cin >> w;
> // الطريقة التقليدية
> if (w % 2 == 0) cout << "Even" << "\n";
> // الطريقة الأسرع والأشيك
> if ((w & 1) == 0) cout << "Even" << "\n";
> ```

> ## 4) Extract Last Digit (استخراج الخانة الأخيرة)

> **القانون:**

> [!TIP]

>

> [!WARNING]

>

>

> ## 5) Intersection of Two Intervals (تقاطع فترتين)
> لحساب فترة التقاطع بين أي فترتين $[L_1, R_1]$ و $[L_2, R_2]$:
> 
> • **بداية التقاطع (Start):** هي أكبر رقم في البدايات $\rightarrow$ `max(L1, L2)`
> • **نهاية التقاطع (End):** هي أصغر رقم في النهايات $\rightarrow$ `min(R1, R2)`
> 

> [!WARNING] 
> شرط التقاطع (Crucial Condition)
> التقاطع بيكون موجود **فقط** لو كانت البداية أصغر من أو تساوي النهاية (`start <= end`). لو العكس، يبقى الفترتين مش متقاطعين أصلاً
>

>[!CAUTION]
> **C++ Template:**
> ```cpp
> int start = max(L1, L2);
> int end = min(R1, R2);
> if (start <= end) cout << start << " " << end << "\n";
> else cout << -1 << "\n";
> ```

> ## 6) Triangle Inequality (متباينة المثلث)
> للتحقق من إمكانية تكوين مثلث (مساحته الموجبة) من 3 أضلاع $A, B, C$:
> **يجب أن يكون مجموع أصغر ضلعين أكبر تماماً من الضلع الأطول.**
>

> [!TIP] 
> **CP Pro Trick: Calculating the Gap**
> إذا لم تكن الأضلاع تكون مثلثاً، فإن أقل قيمة يجب إضافتها لأحد الأضلاع الصغيرة لغلق المثلث هي:
> $Increment = MaxSide - SumOfOtherTwo + 1$
> 

>[!CAUTION] 
> **C++ Template:**
> ```cpp
> int max_side = max({a, b, c});
> int sum_others = (a + b + c) - max_side;
> 
> if (sum_others > max_side) cout << 0 << "\n"; // Valid Triangle
> // Minimum addition needed to make sum_others > max_side
> else cout << (max_side - sum_others) + 1 << "\n"; 
> ```

> لحساب مجموع الأعداد المتتالية من 1 إلى $N$ دون استخدام الحلقات التكرارية (Loops):
> 
> **القانون الرياضي:**
> $Sum = \frac{N \times (N + 1)}{2}$
>

> [!WARNING]
> **CP Trap: Time Limit & Overflow**
>
> • إذا كانت $N \ge 10^8$، استخدام الـ `for loop` سيؤدي حتماً لـ **TLE**.
> • ناتج عملية الضرب $N \times (N+1)$ قد يتخطى $10^{18}$، لذا يجب استخدام `long long` دائمًا.
>

> [!CAUTION]
**C++ Template:**
> ```cpp
> int n;
> cin >> n;
> // Time Complexity O(1)
> long long total_sum = (n * (n + 1)) / 2;
> cout << total_sum << "\n";
> ```

> ## 9) Sum of Two Equals the Third (رقم يساوي مجموع الآخرين)
> للتحقق مما إذا كان أحد الأرقام الثلاثة هو حاصل جمع الرقمين الآخرين:
> 
> **القانون الرياضي:**
> $Largest = (A + B + C) - Largest$
> 

> [!TIP] 
> **CP Pro Tip: The Max-Sum Logic**
>
> بدلاً من تجربة كل التباديل (Permutations) مثل `a + b == c`, `b + c == a` إلخ، ابحث عن الرقم الأكبر واطرحه من المجموع الكلي. إذا كان الناتج يساوي الرقم الأكبر، فالشرط محقق.
> 

> [!CAUTION] 
> **C++ Template:**
> ```cpp
> int a, b, c;
> cin >> a >> b >> c; 
> int largest = max({a, b, c});
> int sum_others = (a + b + c) - largest; 
> if (largest == sum_others) cout << "YES\n";
> else cout << "NO\n";
> ```

> ## 10) Average 2 (المتوسط الموزون)
> لحساب المتوسط الموزون لدرجات طالب لما يكون كل مادة ليها "ثقل" أو وزن (Weight) مختلف عن التانية:
>
>**القانون الرياضي:**
>
> $$MEDIA = \frac{\sum (x_i \cdot w_i)}{\sum w_i} = \frac{(Value_A \times Weight_A) + (Value_B \times Weight_B) + (Value_C \times Weight_C)}{\text{Total Weights}}$$
>

> [!TIP]
> **CP Pro Tip: Weighted Average Logic**
>
> في المتوسط الموزون، إحنا مش بنقسم على عدد المواد، إحنا بنقسم على "مجموع الأوزان" (2 + 3 + 5 = 10). وخليك فاكر دايماً إن التعامل مع الكسور بيحتاج `double` وتظبط الدقة بـ `setprecision(1)` عشان تطلع الناتج بخانة عشرية واحدة زي ما الـ Judge طالب بالظبط.
>

> [!CAUTION] 
> **C++ Template :**
> ```cpp
> double A, B, C;
> cin >> A >> B >> C;
> // بنضرب كل درجة في وزنها ونقسم على مجموع الأوزان (10)
> double media = ((A * 2) + (B * 3) + (C * 5)) / 10.0;
> // الطباعة بالتنسيق المطلوب "MEDIA = X.X"
> cout << "MEDIA = " << fixed << setprecision(1) << media << endl;
> ```

> ## 11) Calculating Function (دالة المجموع المتذبذب)
> لحساب ناتج متسلسلة حسابية تتأرجح فيها الإشارات بين السالب والموجب بالشكل: 
>  $f(n) = -1 + 2 - 3 + 4 - \dots + (-1)^n n$
> 
> **القانون الرياضي:**
> 
> • إذا كان $n$ **زوجي**: $f(n) = \frac{n}{2}$
>
> • إذا كان $n$ **فردي**: $f(n) = -\frac{(n + 1)}{2}$
>

> [!TIP] 
> **CP Pro Tip: Avoid O(N) Loops**
>
> بما أن $n$ قد تصل إلى $10^{15}$، لا يمكنك استخدام حلقة تكرارية (`for loop`) لأنها ستؤدي إلى **Time Limit Exceeded (TLE)**. الحل هو استخدام الملاحظة الرياضية (Pattern Recognition) للوصول للناتج في زمن $O(1)$. أيضاً تأكد من استخدام `long long` لتفادي الـ **Overflow**.
>

> [!CAUTION] 
> **C++ Template:**
> ```cpp
> long long n;
> cin >> n;
> 
> if (n % 2 == 0) cout << n / 2 << endl;
> else cout << -(n + 1) / 2 << endl;
> 
> // Ternary Operator:
> // cout << (n % 2 == 0 ? n / 2 : -(n + 1) / 2) << endl;
> ```

> ## 12) Cyclic Consumption (الاستهلاك الدوري)
>  للتحقق من الشخص الذي سينفد منه المورد (مثل الشامبو) في نظام استهلاك دوري يعيد نفسه بترتيب محدد.
>
> **القانون الرياضي:**
>
> بدلاً من إنقاص المورد في حلقة تكرارية (`while V >= 0`)، نختصر العملية باستخدام باقي القسمة (Modulo) على مجموع الاستهلاك الكلي في الدورة الواحدة: $V_{rem} = V \pmod{A + B + C}$
>

> [!IMPORTANT]
**بعد ذلك نختبر الترتيب:**
> 1. إذا كان $V_{rem} < A \rightarrow$ الشخص الأول (F).
> 2. إذا كان $V_{rem} < (A + B) \rightarrow$ الشخص الثاني (M).
> 3. غير ذلك $\rightarrow$ الشخص الثالث (T).
>

> [!TIP] 
> **CP Pro Tip: The Modulo Shortcut**
> 
> استخدام `%` يوفر عليك محاكاة مئات الدورات (Cycles) التي يستهلك فيها الجميع الشامبو بالكامل. هذه الطريقة تجعل الحل يعمل في زمن ثابت $O(1)$ مهما كانت قيمة $V$ كبيرة، وهي الطريقة المثالية للتعامل مع "الأنماط المتكررة".
>

> [!CAUTION] 
>**C++ Template:**
> ```cpp
> V %= (A + B + C);
> if (V < A) cout << "F" << endl;
> else if (V < (A + B)) cout << "M" << endl;
> else cout << "T" << endl;
> ```

> ## 13) Tricky Sum (المجموع المشروط بقوى العدد 2)
>  لحساب مجموع الأرقام من $1$ إلى $n$ مع تحويل إشارة الأرقام التي تمثل "قوى العدد 2" ($1, 2, 4, 8, \dots$) من موجب إلى سالب.
> 
> **القانون الرياضي:**
> 
> 1. احسب المجموع الكلي للأرقام من $1$ إلى $n$:
> $TotalSum = \frac{n \times (n + 1)}{2}$
> 2. احسب مجموع قوى العدد 2 التي هي $\le n$:
> $SumPowersOfTwo = \sum 2^i \quad \text{where } 2^i \le n$
> 3. اطرح ضعف مجموع القوى من المجموع الكلي (لأننا نريد إزالتها كقيمة موجبة وإضافتها كقيمة سالبة): $Result = TotalSum - (2 \times SumPowersOfTwo)$
>

> [!TIP]
> **CP Pro Tip: Binary Growth Speed**
>
> بدلاً من المرور على جميع الأرقام من $1$ إلى $n$ (وهو ما سيسبب **TLE** لأن $n$ تصل لـ $10^9$)، تذكر أن قوى العدد 2 تنمو بسرعة كبيرة جداً. يوجد حوالي 30 قوة فقط للعدد 2 أقل من مليار ($10^9$). لذا، استخدم حلقة تكرارية تبدأ من $1$ وتتضاعف (`i *= 2`) لحساب مجموع القوى في زمن $O(\log n)$.
>

> [!CAUTION]
> **C++ Template:**
> ```cpp
> long long n;
> cin >> n;
> 
> // 1. المجموع الكلي باستخدام القانون
> long long total_sum = n * (n + 1) / 2;
> // 2. حساب مجموع قوى الـ 2
> long long sum_powers = 0;
> for (long long i = 1; i <= n; i *= 2) sum_powers += i;
> 
> // 3. الناتج النهائي
> long long result = total_sum - (2 * sum_powers);
> cout << result << endl;
> ```

> ## 14) Buy a Shovel (شراء الجاروف بدون فكة)
>  لإيجاد أقل عدد من السلع التي يمكن شراؤها بحيث يكون المبلغ الإجمالي قابلاً للدفع باستخدام عملات من فئة معينة (10 في هذه الحالة) مع إمكانية استخدام عملة واحدة إضافية من فئة أخرى ($r$).
> 
> **القانون الرياضي:**
> 
> نبحث عن أصغر عدد $t$ يحقق أحد الشرطين:
> 
> 1. $(t \times k) \pmod{10} = 0$ (المبلغ يقبل القسمة على 10 بدون فكة).
> 2. $(t \times k) \pmod{10} = r$ (المبلغ يتبقى منه فئة العملة الوحيدة $r$ بعد دفع العشرات).
>

> [!TIP]
> **CP Pro Tip: The Limit of 10**
>
> بما أننا نتعامل مع "باقي القسمة على 10"، فإن أي رقم $t$ أكبر من 10 سيعيد تكرار نفس الآحاد التي ظهرت في الأرقام من 1 إلى 10 (مثلاً آحاد $11 \times k$ هي نفس آحاد $1 \times k$). لذلك، الحل دائماً مضمون وموجود في نطاق بسيط جداً من 1 إلى 10.
>

> [!CAUTION] 
> **C++ Template:**
>```cpp
> int k, r;
> cin >> k >> r;
> 
> for (int t = 1; t <= 10; ++t) {
>     int total_price = t * k;
>     // r التحقق هل المبلغ ينتهي بصفر أو ينتهي بالرقم
>     if (total_price % 10 == 0 || total_price % 10 == r) {
>       cout << t << endl;
>       break;
>    }
> }
>```

> ## 15) Prime Divisibility Constraint (شرط الأولية والقابلية للقسمة)
>  للتحقق مما إذا كان هناك رقم $x$ في النطاق من $1$ إلى $n$ يحقق شرطين معاً: أن يكون **أولياً (Prime)** وفي نفس الوقت **يقبل القسمة على 13**.
> 
> **القانون الرياضي:**
> 
> الرقم الوحيد في الكون الذي يعتبر عدداً أولياً ويقبل القسمة على 13 هو الرقم **13** نفسه.
> 
> • أي رقم آخر يقبل القسمة على 13 (مثل 26، 39، 52...) ليس أولياً لأنه يمتلك قواسم أخرى غير نفسه والواحد.
>     
> • بناءً عليه، الشرط يتحقق فقط إذا كان الرقم 13 موجوداً ضمن النطاق المعطى، أي: $n \ge 13$.
>     

> [!TIP] 
> **CP Pro Tip: Logic Simplification**
> 
> المسائل التي تدمج بين "الأعداد الأولية" و"القابلية للقسمة" غالباً ما تهدف لاختبار فهمك لتعريف العدد الأولي. القاعدة العامة: إذا كان $P$ عدداً أولياً، فإن العدد الوحيد الذي يقبل القسمة على $P$ ويكون أولياً في نفس الوقت هو $P$ فقط. هذا يحول المسألة من بحث معقد عن الأعداد الأولية إلى مجرد مقارنة بسيطة $O(1)$.
>

> [!CAUTION] 
> **C++ Template:**
> ```cpp
> long long n;
> cin >> n;
> 
> // بما أن 13 هو الرقم الوحيد الذي يحقق الشرط
> if (n >= 13) cout << "YES" << endl;
> else cout << "NO" << endl;
> ```

> ## 16) Sorting Three Numbers (ترتيب ثلاثة أرقام)
>  لترتيب ثلاثة أرقام تصاعدياً بدون استخدام دوال الترتيب الجاهزة (Sorting Algorithms) أو تعقيدات الشروط المتداخلة.
> 
> **القانون الرياضي:**
> 
> 1. **الأصغر (Smallest):** هو $min(A, B, C)$.
> 2. **الأكبر (Largest):** هو $max(A, B, C)$.
> 3. **الأوسط (Middle):** هو حاصل طرح الأصغر والأكبر من المجموع الكلي: $Middle = (A + B + C) - (Smallest + Largest)$.
>     

> [!TIP] 
> **CP Pro Tip: The Sum Elimination Trick**
> 
> هذه الخدعة هي الأسرع لترتيب **3 عناصر فقط** في زمن $O(1)$. بدلاً من كتابة 6 احتمالات (Permutations) باستخدام `if-else` أو استخدام `std::sort` التي تستهلك وقتاً أطول قليلاً، نستخدم المجموع الكلي لاستنتاج العنصر المفقود. هذه الطريقة تقلل من الأخطاء المنطقية وتجعل الكود أكثر نظافة.
>

> [!CAUTION] 
> **C++ Template:**
> ```cpp
> long long a, b, c;
> cin >> a >> b >> c;
> long long smallest = min({a, b, c});
> long long largest = max({a, b, c});
>
> // استنتاج الرقم الثالث بطرح المعلوم من المجموع الكلي
> long long middle = (a + b + c) - (smallest + largest);
>
> cout << smallest << " " << middle << " " << largest << endl;
> ```

> ## 17) Sum of k Distinct Odds (مجموع k أعداد فردية مختلفة)
>  لتحقق مما إذا كان بالإمكان كتابة الرقم $n$ كحاصل جمع $k$ من الأعداد الفردية الموجبة **والمختلفة**.
> 
> **القواعد الرياضية:**
> 
> 1. **الحد الأدنى للمجموع:** أصغر مجموع لـ $k$ أعداد فردية مختلفة هو ($1 + 3 + 5 + \dots + 2k-1$)، وناتج هذه المتسلسلة دائماً يساوي $k^2$. لذا يجب أن يكون $n \ge k^2$.
> 2. **شرط الزوجية والفردية (Parity):** مجموع $k$ من الأعداد الفردية يكون زوجياً إذا كانت $k$ زوجية، وفردياً إذا كانت $k$ فردية. لذا يجب أن يتشابه $n$ و $k$ في النوع (كلاهما زوجي أو كلاهما فردي).
>     
>

> [!TIP] 
> **CP Pro Tip: The Square of Odds**
>
> تذكر دائماً هذه القاعدة الرياضية: "مجموع أول $k$ أعداد فردية هو $k^2$". هذه الملاحظة تحول المسألة من محاولة تجربة الأرقام (Backtracking/Greedy) إلى مجرد معادلة رياضية بسيطة تنفذ في زمن $O(1)$.
>

> [!CAUTION] 
> **C++ Template:**
> ```cpp
> long long n, k;
> cin >> n >> k;
> 
> // الشرط الأول: المجموع يجب أن يكون أكبر من أو يساوي أصغر مجموع ممكن
> // الشرط الثاني: يجب أن يكون للرقمين نفس الخاصية (زوجي/فردي)
> if (n >= k * k && (n % 2 == k % 2)) cout << "YES" << endl;
> else cout << "NO" << endl;
> ```

> [!NOTE] 
> ## 18) Race Ranking (عدّ المتسابقين في المقدمة)
>  لحساب عدد العناصر التي قيمتها أكبر من قيمة مرجعية معينة (مثل حساب كم شخص ركض مسافة أطول من الشخص الأول).
> 
> **القانون الرياضي:**
> 
> العملية هي "عد شرطي" (Conditional Counting):
> 
> $$Count = [b > a] + [c > a] + [d > a]$$
> 
> حيث $[condition]$ تساوي $1$ إذا تحقق الشرط و $0$ إذا لم يتحقق.
>

> [!TIP] 
> **CP Pro Tip: Boolean Arithmetic**
> 
> في لغة ++C، ناتج العمليات المنطقية (مثل `a < b`) يعامل معاملة الرقم `1` للصح و `0` للخطأ. يمكنك جمع هذه العمليات مباشرة بدلاً من استخدام جمل `if` المتعددة أو العدادات (`counters`). هذا الأسلوب يسمى **Branchless Programming** وهو يجعل الكود أقصر وأسرع في التنفيذ.
>

> [!CAUTION] 
> **C++ Template:**
>```cpp
> int a, b, c, d;
> cin >> a >> b >> c >> d;
> 
> // جمع نواتج المقارنات مباشرة كقيم رقمية
> int result = (a < b) + (a < c) + (a < d);
>
> cout << result << endl;
> ```

> [!NOTE] 
> ## 19) Summation Identities (الفرق بين مربع المجموع ومجموع المربعات)
>  لحساب ناتج طرح مجموع مربعات الأرقام من مربع مجموع الأرقام (وهي مسألة شهيرة في الرياضيات تُعرف أحياناً بـ Project Euler Problem 6).
> 
> **القوانين الرياضية:**
> 
> 1. **مجموع الأرقام الطبيعية (Sum of n numbers):** $S = \frac{n(n+1)}{2}$
> 2. **مربع المجموع (Square of Sum):** $S_2 = S^2 = (\frac{n(n+1)}{2})^2$
> 3. **مجموع المربعات (Sum of Squares):** $S_1 = \frac{n(n+1)(2n+1)}{6}$
> 4. **الناتج النهائي:** $F(n) = S_2 - S_1$
>

> [!TIP]
> **CP Pro Tip: The 0.5s Time Constraint**
> 
> في هذه المسألة، عدد حالات الاختبار ($t$) كبير جداً ($10^6$) بينما قيمة $n$ صغيرة نسبياً ($10^4$). استخدام القوانين الرياضية المباشرة يجعل التعقيد $O(1)$ لكل حالة اختبار، وهو الحل الوحيد الذي سيعبر من حاجز الـ 0.5 ثانية. إذا كانت $n$ أكبر من ذلك بكثير، يجب الحذر من الـ **Overflow** عند حساب $S^2$.
>

> [!CAUTION] 
> **C++ Template:**
>```cpp
> long long n;
> cin >> n;
> 
> // حساب مربع مجموع الأرقام (S2)
> long long sum = (n * (n + 1)) / 2;
> long long s2 = sum * sum;
> 
> // حساب مجموع مربعات الأرقام (S1)
> long long s1 = (n * (n + 1) * (2 * n + 1)) / 6;
>
> cout << s2 - s1 << endl;
> ```

> [!NOTE] 
> # 20) Distance to Origin (المسافة إلى نقطة الأصل)
>  للمقارنة بين بُعد نقطتين عن مركز الإحداثيات $(0,0)$ لتحديد أيهما أقرب.
> 
> **القانون الرياضي:**
> 
> المسافة الإقليدية (Euclidean Distance) بين نقطة $(x, y)$ ونقطة الأصل هي $\sqrt{x^2 + y^2}$. للمقارنة بين مسافتين، يكفي مقارنة المربع لتجنب الجذور:
> 
> $$D^2 = x^2 + y^2$$
> 
> - إذا كان $D_R^2 < D_W^2 \rightarrow$ Russo هو الأقرب.
> - إذا كان $D_W^2 < D_R^2 \rightarrow$ Wil هو الأقرب.
> - إذا تساويا $\rightarrow$ النتيجة "Empate".
>     

> [!TIP] 
> **CP Pro Tip: Avoid Square Roots**
>
> في مسائل الهندسة والمقارنات، تجنب دائماً استخدام دالة `sqrt()` إلا للضرورة القصوى. التعامل مع الأرقام الصحيحة (Integer Arithmetic) أدق وأسرع، لأن الجذور قد تسبب أخطاء بسيطة في الدقة (Precision Errors) تؤدي لـ **Wrong Answer**.
>

> [!CAUTION]
> **C++ Template:**
> ```cpp
> long long xr, yr, xw, yw;
> cin >> xr >> yr >> xw >> yw;
> 
> // حساب مربع المسافة لكل لاعب
> long long russo_dist = (xr * xr) + (yr * yr);
> long long wil_dist = (xw * xw) + (yw * yw);
> 
> if (russo_dist < wil_dist) cout << "Russo" << endl;
> else if (wil_dist < russo_dist) cout << "Wil" << endl;
> else cout << "Empate" << endl;
> ```

> [!NOTE] 
> ## 21) Range Sum (مجموع الأرقام في نطاق محدد)
>  لحساب مجموع الأرقام المتتالية في نطاق يبدأ من $L$ وينتهي عند $R$ (شاملاً الطرفين).
>
> **القانون الرياضي:**
>
> يعتمد الحل على قانون المتسلسلة الحسابية (Arithmetic Progression):
> 1. **عدد الأرقام (Count):** $n = R - L + 1$
> 2. **المجموع:** $Sum = \frac{n \times (L + R)}{2}$
> أو بطريقة أخرى: (مجموع الأرقام من $1$ إلى $R$) - (مجموع الأرقام من $1$ إلى $L-1$).
>

> [!TIP]
> **CP Pro Tip: Big Numbers & O(1)**
> 
> بما أن القيم قد تصل إلى $10^9$، فإن استخدام حلقة تكرارية (`for loop`) سيؤدي حتماً إلى **TLE** لأن العملية ستستغرق وقتاً طويلاً جداً. استخدام القانون الرياضي يحل المسألة في زمن **$O(1)$**. انتبه دائماً لاستخدام `long long` لأن ناتج ضرب $(10^9 \times 10^9)$ سيتخطى حدود الـ `int` ويسبب **Overflow**.
>

> [!CAUTION] 
> **C++ Template:**
> ```cpp
> long long L, R;
> cin >> L >> R;
> 
> // عدد العناصر في النطاق
> long long n = R - L + 1;
>
> // تطبيق قانون المجموع
> long long sum = (n * (L + R)) / 2;
> 
> cout << sum << endl;
> ```
